[모듈 2: 데이터 그룹화] HashMap 기초 - 이름표가 붙은 마법의 주머니

1. 💡 눈높이 비유 (The Analogy)

💡 목욕탕 아주머니의 열쇠와 옷 바구니!

배열이나 ArrayList는 물건을 보관할 때 방 번호(0, 1, 2...)만 기억했습니다. "125번째 서랍에 여권이 들어있어!" 라고 외우려니 사람이 쓰기엔 너무 헷갈립니다.

HashMap은 '단어가 적힌 이름표(Key)'와 '실제 물건(Value)'이 세트로 들어가는 스마트 주머니입니다.

  • **Key (열쇠/이름표):** 오직 한 개만 존재하는 고유한 이름표입니다. (예: "내여권", "엄마번호")
  • **Value (물건/데이터):** 그 이름표에 줄로 묶여 딸려오는 실제 데이터입니다. (예: 여권실물, 010-1234-5678)
  • 이제 방 번호는 잊으세요! "지갑 꺼내줘!" 하면 지갑이 나오고, "현관비밀번호 꺼내줘!" 하면 비밀번호가 나옵니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① Key-Value 한 쌍 ( <K, V> )
    👉 "생성할 때 HashMap<String, Integer> 처럼 두 가지를 정합니다. 앞은 '이름표의 재질(문자열)'이고 뒤는 '내용물의 종류(숫자)'입니다. (예: 이름표 '국어' ➔ 점수 '90')"
  • ② 넣기 ( .put() )
    👉 "ArrayList는 무식하게 .add()로 던져 넣었지만, HashMap은 .put("이름표", 물건) 처럼 항상 이름표와 물건을 같이 제출해야 주머니에 들어갑니다."
  • ③ 꺼내기 ( .get() )
    👉 "꺼낼 때는 방 번호가 아니라 .get("이름표") 처럼 열쇠(Key)를 제시해야만 보관해둔 물건을 받을 수 있습니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

친구들의 나이를 헷갈리지 않게 '이름표-나이' 세트로 저장하는 연락처 앱을 만들어 볼까요?

// 사용하려면 반드시 import 필요!
import java.util.HashMap;

public class MapTest {
    public static void main(String[] args) {
        
        // 1️⃣ HashMap 생성 (이름표: String 문자, 내용물: Integer 숫자)
        HashMap<String, Integer> ages = new HashMap<>();
        
        // 2️⃣ 데이터 넣기 (put) - <Key, Value> 세트
        ages.put("철수", 20);
        ages.put("영희", 22);
        ages.put("민수", 21);
        
        // 3️⃣ 데이터 꺼내보기 (get) - 방 번호(0)가 아닌 이름표(Key)로 꺼냄!
        System.out.println("영희의 나이는? " + ages.get("영희") + "살");
        
        // 4️⃣ 데이터 변경하기 (같은 Key에 put을 또 하면 덮어쓰기 됨)
        // 영희가 한 살 지났다면? "영희"라는 열쇠로 새 나이 23을 넣습니다!
        ages.put("영희", 23);
        System.out.println("해를 넘긴 영희 나이: " + ages.get("영희") + "살");
        
        // 5️⃣ 데이터 지우기 (remove) - 키 값만 주면 가방에서 꺼내서 버립니다.
        ages.remove("민수"); 
        System.out.println("민수가 이민을 갔습니다. 현재 연락처: " + ages);
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 앗! 중복 열쇠 (Key 덮어쓰기): ArrayList는 "사과"를 여러 번 넣어도 각각 저장되지만, HashMap의 Key는 절대 중복될 수 없습니다. 이미 "영희"가 있는데 .put("영희", 99)를 해버리면 기존 영희의 데이터는 삭제되고 99로 무자비하게 덮어쓰여집니다!
  2. 🚨 없는 열쇠로 열기 (null 반환): ages.get("동건") 처럼 주머니에 넣지도 않은 이름표를 달라고 하면, 에러는 나지 않지만 텅 비었다는 뜻인 null 이라는 유령 값을 던져줍니다. 나중에 계산하려다 터질 수 있으니 조심하세요!
  3. 🚨 순서의 배신: 내가 "철수", "영희", "민수" 순으로 다이어리에 적어 넣어도, System.out.println(ages) 로 확인해보면 컴퓨터 마음대로 마구 뒤섞여 저장되어 있습니다! HashMap은 속도를 위해 바구니 안에 아무렇게나 짐을 던져놓는 구조(순서 보장 안 됨)이기 때문입니다.

4. 📝 스스로 챌린지 (Mini Challenge)

게임 속 상점의 물건과 가격표를 HashMap으로 만들어 봅시다!

📝 미션: 상점 가격표 관리자

1. 문자열 이름표(물건 이름)와 정수형 내용물(가격)이 담기는 HashMap 변수 shop을 만드세요.
2. put을 이용해 "포션"은 500원, "해독제"는 300원으로 저장하세요.
3. 내가 "포션"과 "해독제"를 각각 하나씩 샀을 때, 총 지불해야 할 총 금액(totalPrice)을 get 기능으로 꺼내어 계산하고 화면에 출력해 보세요!

👉 힌트 & 정답(클릭해서 확인하세요)
정답 코드: 
import java.util.HashMap;

public class Main {
    public static void main(String[] args) {
        HashMap<String, Integer> shop = new HashMap<>();
        
        // 가격 세팅
        shop.put("포션", 500);
        shop.put("해독제", 300);
        
        // 이름표로 장바구니 가격 조회하기
        int potionPrice = shop.get("포션");
        int antidotePrice = shop.get("해독제");
        
        int totalPrice = potionPrice + antidotePrice;
        
        System.out.println("총 지불해야 할 금액: " + totalPrice + "원");
    }
}
