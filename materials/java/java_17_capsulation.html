[모듈 3: 클래스와 객체] 캡슐화(Encapsulation) - 은행 금고와 ATM 기계

1. 💡 눈높이 비유 (The Analogy)

💡 은행 금고를 숨기고, ATM만 열어두자!

만약 은행 계좌 클래스의 잔액(balance) 접근이 누구에게나 허용된다면 어떨까요?
해커(다른 클래스)가 myAccount.balance = 99999999; 라고 자기 마음대로 잔고를 조작해버릴 수 있습니다. 상상만 해도 끔찍하죠?

캡슐화(Encapsulation)는 이렇게 민감한 데이터(필드)를 아무도 못 건드리게 꽁꽁 숨기고(private), 오직 철저한 검사를 거친 안전한 창구(Getter/Setter 메서드)를 통해서만 접근하도록 통제하는 기술입니다.

  • **private 필드 = 철창이 쳐진 은행 금고:** 오직 그 은행(클래스 내부)에서만 열 수 있습니다. 외부 접근 절대 차단!
  • **public Getter/Setter = 은행 창구 직원 (ATM):** 고객이 비밀번호를 제대로 쳤는지, 마이너스 금액을 입금하진 않는지 깐깐하게 검사한 뒤 통과시켜 줍니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 접근 제어자 private
    👉 "자물쇠입니다! 변수 이름 앞에 private int age; 라고 달아두면, 나(현재 클래스) 말고 다른 어떤 누구도 이 변수에 접근하거나 값을 수정할 수 없게 철벽 방어를 합니다."
  • ② Setter (세터 - 값을 저장/수정하는 대리인)
    👉 "외부에서 금고(필드)에 값을 넣고 싶을 때 부르는 직원의 이름(관례)입니다. 보통 set변수명() 모양을 가집니다. 여기서 '나이가 0살 미만인가요?' 같은 유효성 검사를 진행합니다."
  • ③ Getter (게터 - 값을 읽어오는 대리인)
    👉 "외부에서 금고(필드)의 값을 구경하고 싶을 때 부르는 직원의 이름입니다. get변수명() 모양을 가지며, 안전하게 복사본만 보여주고 진짜 데이터는 지켜냅니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

나이를 절대로 음수(-)로 설정할 수 없는 완벽한 사람(Person) 클래스를 만들어 봅시다!

class Person {
    // 1️⃣ 필드를 무조건 private으로 '은닉'합니다! 외부 접근 차단.
    private String name;
    private int age;
    
    // 2️⃣ 이름 Setter / Getter
    public void setName(String n) {
        name = n;
    }
    public String getName() {
        return name;
    }
    
    // 3️⃣ 나이 Setter (문지기 역할 톡톡히 하기!)
    public void setAge(int a) {
        // 무조건 넣는 게 아니라 거름망(조사)을 거칩니다.
        if (a < 0) {
            System.out.println("🚨 경고: 나이는 0보다 작을 수 없습니다! (기본값 0 설정)");
            age = 0;
        } else {
            age = a;
        }
    }
    
    // 나이 Getter
    public int getAge() {
        return age;
    }
}

public class EncapsulationMain {
    public static void main(String[] args) {
        
        Person p = new Person();
        
        // ❌ 에러 발생: p.age = -50; 
        // (private이라 안 보여서 조작 불가! 해킹 철벽 방어!)
        
        // ✅ 오직 허락된 창구(Setter)를 통해서만 값을 넣습니다.
        p.setName("유재석");
        p.setAge(-50); // 문지기가 막아내고 0으로 세팅해줍니다.
        
        // 값을 꺼내 볼 때도 허락된 창구(Getter)를 통해서!
        System.out.println(p.getName() + "님의 나이는 " + p.getAge() + "살 입니다.");
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 귀찮다고 전부 public으로 열어두기: 공부할 땐 편하지만 실무에서 그렇게 짰다간 사수한테 엄청나게 혼납니다. 객체지향의 기본 철칙은 "필드(변수)는 무조건 private, 메서드는 보통 public" 입니다!
  2. 🚨 Getter에서 값 반환 안 해주기: getAge() 메서드는 내 나이가 몇 살인지 물어보는 창구입니다. 따라서 반드시 int getAge() {...} 처럼 리턴 타입을 적고, 끝에 return age; 를 써서 값을 돌려보내 줘야 합니다.
  3. 🚨 IDE(VS Code)의 자동 완성 꿀팁: 현업에선 Getter/Setter를 일일이 수작업으로 치지 않습니다! VS Code나 이클립스에서 필드만 만들어 놓고 우클릭(또는 Quick Fix) 메뉴를 보면 "Generate Getters and Setters"라는 마법의 버튼이 다 만들어 줍니다.

4. 📝 스스로 챌린지 (Mini Challenge)

회원가입 비밀번호를 안전하게 관리하는 Member 클래스를 캡슐화해 봅시다!

📝 미션: 철통 보안 비밀번호 만들기

1. Member 클래스를 생성하고 private String password; 필드를 만드세요.
2. setPassword(String pw) 세터를 만드세요. 단! 입력된 pw의 길이가 4글자 미만이면 "비밀번호가 너무 짧습니다!"라고 출력하고 password에 저장하지 마세요.
(글자 수 제한 힌트: pw.length() < 4)
3. 본부석에서 Member 객체를 만들고 setPassword("12") 와 setPassword("12345")를 각각 시도해 보세요.

👉 힌트 & 정답(클릭해서 확인하세요)
정답 코드: 
class Member {
    private String password;
    
    public void setPassword(String pw) {
        // 들어온 데이터 유효성 검사! (캡슐화의 꽃)
        if (pw.length() < 4) {
            System.out.println("🚨 실패: 비밀번호가 너무 짧습니다! (최소 4자 이상)");
        } else {
            password = pw;
            System.out.println("✅ 비밀번호가 성공적으로 변경되었습니다.");
        }
    }
    
    public String getPassword() {
        return password;
    }
}

public class Main {
    public static void main(String[] args) {
        Member m = new Member();
        m.setPassword("12");    // 너무 짧아서 거부됨
        m.setPassword("12345"); // 안전하게 저장됨
    }
}
