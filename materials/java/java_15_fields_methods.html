[모듈 3: 클래스와 객체] 속성(필드)과 동작(메서드) - 객체에 데이터와 기능 넣기

1. 💡 눈높이 비유 (The Analogy)

💡 게임 캐릭터 분석하기! (명사와 동사)

우리가 게임에서 마법사 캐릭터를 만들었다고 상상해 봅시다. 이 마법사는 혼자서 완성되지 않습니다. 이 캐릭터를 구성하는 두 가지 중요한 기둥이 있습니다.

  • **필드(Field) = 명사 / 상태 / 스펙**
    마법사의 이름, 체력(HP), 레벨, 남은 마나(MP) 같은 것들입니다. 캐릭터가 "가지고 있는 정보" 혹은 "상태값"을 뜻합니다.
  • **메서드(Method) = 동사 / 동작 / 기능**
    마법사가 달린다(), 파이어볼을 쏜다(), 휴식하여 체력을 회복한다() 같은 것들입니다. 캐릭터가 스스로 할 수 있는 "능력"이나 "행동"을 뜻합니다.

객체(캐릭터)는 이처럼 필드(정보)와 메서드(행동)가 하나로 찰떡처럼 뭉쳐진 완전체입니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 필드 선언 ( 전역 변수 )
    👉 "클래스의 중괄호 {} 바로 안쪽에 선언된 변수를 '필드' 또는 '멤버 변수'라고 합니다. 잠깐 쓰다 버리는 지역변수와 달리, 객체가 살아있는 동안 함께 영원히 유지되는 소중한 데이터입니다."
  • ② 메서드 선언 ( 리턴타입 이름() {} )
    👉 "객체가 할 수 있는 행동을 정의합니다. void attack() { ... } 처럼 만들어 두면, 나중에 캐릭터.attack(); 한 번으로 공격 기능을 재사용할 수 있습니다."
  • ③ 도트 연산자 ( . )
    👉 "만들어진 객체에게 접근하는 만능 리모컨 버튼입니다! hero.hp = 100; (필드 정보 건드리기), hero.run(); (메서드 행동 시키기) 처럼 이름 뒤에 점(.)을 찍어 모든 걸 조종합니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

전사 캐릭터의 설계도를 꼼꼼히 짜고, 그 안에서 필드와 메서드가 어떻게 연동되는지 봅시다.

// 1️⃣ 설계도(클래스) 영역
class Warrior {
    // --- [필드: 상태/명사] ---
    String name;  // 캐릭터 이름
    int hp = 100; // 체력 (기본값 100을 줘서 태어남)
    int level = 1; // 레벨
    
    // --- [메서드: 행동/동사] ---
    // 1. 공격 기능
    void attack() {
        System.out.println(name + "이(가) 호쾌하게 칼을 휘두릅니다! 슉!");
    }
    
    // 2. 피해받기 기능 (매개변수를 통해 데미지를 받음!)
    void takeDamage(int damage) {
        hp = hp - damage; // 내 체력(필드)에서 들어온 데미지만큼 깎음
        System.out.println(name + "이(가) " + damage + "의 피해를 입었습니다! (남은 체력: " + hp + ")");
    }
}

// 2️⃣ 실행(본부석) 영역
public class GameMain {
    public static void main(String[] args) {
        
        // 전사 객체 1 생성!
        Warrior knight = new Warrior();
        
        // 점(.)을 찍어 필드에 접근하여 이름표 달아주기
        knight.name = "아서왕"; 
        
        // 점(.)을 찍어 메서드(행동) 호출하기
        knight.attack(); 
        
        // 몬스터에게 30의 데미지를 받았다고 지시 내리기
        knight.takeDamage(30); 
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 메서드 호출 시 괄호() 빼먹기: 필드(체력)를 볼 때는 knight.hp 가 맞지만, 행동(공격)을 명령할 때는 knight.attack() 처럼 뒤에 반드시 괄호 ()를 붙여야 "이 행동을 실행하라!"는 뜻이 됩니다.
  2. 🚨 지역변수 vs 전역변수(필드) 착각: 메서드 안에 쓰인 int damage 같은 변수는 메서드가 끝나는 순간 공중분해됩니다(지역변수). 하지만 클래스 바로 밑에 적힌 int hp 같은 필드는 객체가 죽기 전까지 계속 기억됩니다! 혼동하지 마세요.
  3. 🚨 초기화 없이 필드 사용: 숫자는 0, 문자열은 null이 기본값으로 들어갑니다. 이름을 지어주지도 않고 attack()을 부르면 "null이(가) 휘두릅니다!" 라며 바보 같은 출력이 나오니 객체를 만든 후 생명을 불어넣어(초기화) 주세요.

4. 📝 스스로 챌린지 (Mini Challenge)

은행 계좌(BankAccount) 클래스를 직접 설계해 봅시다!

📝 미션: 나만의 은행 계좌 조종하기

1. BankAccount 클래스를 만들고, 필드로 String owner(소유자명), int balance(잔액, 기본값 0)를 선언하세요.
2. deposit(int money) 라는 입금 메서드를 만드세요. 이 메서드는 들어온 돈만큼 내 잔액(balance)을 증가시켜야 합니다.
3. 본부석(main)에서 account1 통장을 새로 만들고, 소유자를 본인 이름으로 바꾸세요.
4. deposit(50000) 메서드를 호출해 5만 원을 입금시킨 뒤, "현재 잔액은 50000원입니다." 라고 출력해 보세요.

👉 힌트 & 정답(클릭해서 확인하세요)
정답 코드: 
// 통장 설계도
class BankAccount {
    String owner;
    int balance = 0;
    
    // 입금 기능(메서드)
    void deposit(int money) {
        balance = balance + money;
        System.out.println(money + "원이 입금되었습니다. 띠링!");
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount account1 = new BankAccount();
        account1.owner = "홍길동";
        
        // 5만원 저금통에 넣기!
        account1.deposit(50000);
        
        System.out.println(account1.owner + "님의 현재 잔액은 " + account1.balance + "원입니다.");
    }
}
