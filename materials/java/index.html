☕ 자바 마스터 교육: 0. 시작하기 (환경 세팅)

우리 함께 즐거운 자바(Java)의 세계로 떠나볼까요? 첫 번째 준비 단계입니다!

1. 💡 눈높이 비유 (The Analogy)

💡 우리는 지금부터 '이탈리안 레스토랑'을 열 거예요!

자바(Java)를 배우는 것은 마치 식당을 열고 요리를 시작하는 것과 같습니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

이번 단원에서 가장 중요한 3가지 용어를 기억하세요. "이게 왜 필요한가?"에 초점을 맞췄습니다!

🔑 필수 키워드 3가지
  • ① JDK (Java Development Kit)
    👉 "자바로 글을 쓰고(컴파일), 만든 프로그램을 실행하게 해주는 모든 도구의 묶음이에요."
  • ② 환경 변수 (Environment Variable)
    👉 "컴퓨터에게 '앞으로 내가 자바라고 부르면 여기가서 도구 가져와!' 하고 지도(경로)를 새겨주는 설정입니다."
  • ③ VS Code (Visual Studio Code)
    👉 "MS에서 만든 강력하고 가벼운 코드 편집기입니다. 자바뿐만 아니라 수많은 언어의 '스마트 맞춤형 노트장' 역할을 해요."

3. ⬇️ 자바(JDK) 다운로드 및 설치하기

자바 프로그램을 만들고 실행하기 위해 가장 뼈대가 되는 JDK를 설치합니다.

  • LTS 버전 선택: 구글에 'Java JDK Download' 혹은 'Amazon Corretto'를 검색합니다. 메인 화면에 있는 'Latest LTS Release' (일반적으로 Java 17 또는 21) 버튼을 클릭하여 설치 파일을 다운로드합니다. (*LTS는 가장 안정적이고 오랫동안 지원받는 버전입니다.)
  • 자동 설정(체크박스): 설치를 진행하다 보면 "Set JAVA_HOME variable" (또는 경로 자동 추가) 같은 옵션 체크박스가 나올 때가 있습니다. 이걸 체크하면 아래 복잡한 환경 변수 설정을 알아서 해줍니다!
  • 수동 설정: 혹시 체크박스가 없었거나, 실수로 체크하지 않고 그냥 'Next'만 눌러 설치했다면? 걱정 마세요! 아래의 수동 설정 가이드를 따라가면 됩니다.

4. ️ 환경 변수 수동 설정하기 (5단계 정복!)

설치할 때 경로 자동 추가를 놓쳤다면, 윈도우(Windows)에게 직접 자바 주소를 가르쳐주어야 합니다.

📍 1단계: 자바가 설치된 경로 복사하기

먼저 자바가 내 컴퓨터 어디에 깔려 있는지 주소를 알아야 합니다.

  1. 키보드의 Windows 키 + E를 눌러 파일 탐색기를 엽니다.
  2. C드라이브 > Program Files > Java (또는 Eclipse Adoptium 등 본인이 설치한 자바 폴더)로 들어갑니다.
  3. jdk-버전명으로 되어 있는 폴더 안으로 들어갑니다.
  4. 탐색기 상단의 주소창 부분을 마우스로 클릭한 뒤, 파랗게 블록이 잡힌 그 경로(예: C:\Program Files\Java\jdk-17)를 복사(Ctrl + C)합니다.

📍 2단계: 환경 변수 설정 창 열기

  1. 윈도우 화면 왼쪽 아래의 검색창(돋보기 모양)에 "환경 변수"라고 검색합니다.
  2. 검색 결과 중 "시스템 환경 변수 편집"을 클릭합니다.
  3. '시스템 속성' 창이 뜨면, 오른쪽 아래에 있는 [환경 변수(N)...] 버튼을 클릭합니다.

📍 3단계: JAVA_HOME 변수 만들기

컴퓨터에게 '자바의 집(설치 폴더)이 여기야'라고 알려주는 과정입니다.

  1. 창 아래쪽의 '시스템 변수(S)' 영역에서 [새로 만들기(W)...] 버튼을 클릭합니다.
  2. 다음과 같이 입력합니다:
    - 변수 이름(N): JAVA_HOME (반드시 대문자로 입력해 주세요!)
    - 변수 값(V): 1단계에서 복사해 둔 경로를 붙여넣기(Ctrl + V) 합니다.
  3. [확인]을 누릅니다.

📍 4단계: Path 변수에 추가하기

컴퓨터가 자바 실행 파일(bin 폴더 안의 파일들)을 바로 찾을 수 있게 길을 뚫어주는 과정입니다.

  1. 여전히 '시스템 변수(S)' 영역에 있는 목록 중에서 Path라는 이름의 변수를 찾아 더블클릭합니다. (또는 선택 후 편집 클릭)
  2. 오른쪽의 [새로 만들기(N)] 버튼을 누릅니다.
  3. 목록의 맨 아래에 빈칸이 생기면 %JAVA_HOME%\bin 이라고 정확하게 입력하고 엔터를 칩니다.
  4. 열려있는 모든 창의 확인 버튼을 눌러 창을 전부 닫아줍니다.

📍 5단계: 설정이 잘 되었는지 확인하기 (중요⭐)

  1. 윈도우 검색창에 cmd를 검색하여 명령 프롬프트를 실행합니다.
  2. 검은 창이 뜨면 java -version을 입력하고 엔터를 칩니다. 자바 버전 정보가 쭉 나오면 1차 성공입니다!
  3. 추가로 javac -version을 입력하고 엔터를 칩니다. 역시 버전 정보가 나오면 환경 변수 설정이 완벽하게 끝난 것입니다! 🎉

(주의: 만약 VS Code나 명령 프롬프트가 이미 켜져 있었다면, 방금 한 설정이 반영될 수 있도록 해당 프로그램을 껐다가 다시 켜주세요.)

5. 🛠️ VS Code로 자바 코딩하기 (IDE 세팅)

비싼 도구 없이도, 아주 강력한 무료 에디터인 Visual Studio Code를 통해 자바 프로그램을 작성하고 실행해 보겠습니다!

  • 1단계 (설치): 구글에 "VS Code Download"를 검색하여 공식 홈페이지에서 설치합니다.
  • 2단계 (확장팩 추가): VS Code를 열고 왼쪽 메뉴 중 블록 모양 아이콘(Extensions, 단축키 Ctrl+Shift+X)을 클릭합니다.
  • 3단계 (자바 마법사 소환 🪄): 검색창에 "Extension Pack for Java"를 검색합니다. (Microsoft에서 제공하는 빨간색 커피잔 아이콘입니다.)
  • 4단계 (설치 완료): 'Install' 버튼을 눌러 설치하면, VS Code가 자바 전용 도우미로 변신합니다!

4. 💻 실습 코드 & 상세 분석 (Code Snippet)

VS Code에서 Main.java라는 파일을 하나 만들고, 자바의 가장 표준적인 첫 코드를 작성해 볼까요?

public class Main {
    // 자바 프로그램이 시작될 때 가장 먼저 실행되는 대문입니다 (메인 메서드)
    public static void main(String[] args) {
        
        // 'System.out'은 모니터 화면을, 'println'은 화면에 줄을 바꿔서 글을 찍어준다는 뜻이에요.
        // 주의: 안에 적을 문구는 반드시 쌍따옴표(" ")로 감싸야 합니다.
        System.out.println("Hello, World!");

        // 자바에서는 명령이 끝날 때마다 항상 세미콜론(;)을 마침표처럼 찍어야 한답니다!
        System.out.println("반갑습니다, 자바 초보 마스터입니다!");
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3

이 부분이 핵심이에요! 처음 자바를 접할 때 여기서 에러를 가장 많이 만납니다.

  1. 🚨 대소문자의 헷갈림: System과 String은 반드시 **대문자(S)**로 시작해야 해요! 소문자로 적으면 에러(Error)가 발생합니다.
  2. 🚨 세미콜론 누락: 문장의 끝인 세미콜론(;)을 빼먹어서 빨간 줄이 나타나는 경우가 절반 이상이에요.
  3. 🚨 클래스명과 파일명의 불일치: class Main 이라고 코드를 적었다면, 저장하는 파일 이름도 반드시 Main.java 여야만 합니다!

[모듈 1: 기초 탄탄] 자바의 기본 구조와 화면 출력

1. 💡 눈높이 비유 (The Analogy)

💡 자바 프로그램은 러시아 인형(마트료시카) 같아요!

가장 큰 인형 안에 작은 인형이 있고, 그 안에 또 작은 인형이 있는 장난감을 본 적 있나요? 자바 코드의 구조가 딱 그렇습니다!

  • 가장 큰 껍데기는 **클래스(Class)**라는 집입니다. (모든 코드는 이 집 안에 있어야 해요)
  • 그 집 안에는 가장 중요한 방인 **메인 메서드(Main Method)**가 있습니다. 컴퓨터는 무조건 이 방부터 찾아 들어옵니다.
  • 그 방 안에는 우리가 컴퓨터에게 내리는 구체적인 **명령어**들이 들어 있습니다.

즉, [클래스] ➔ [메인 메서드] ➔ [명령어] 순서로 겹겹이 감싸져 있는 구조랍니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

오늘 가장 처음 마주하게 될 외계어 같은 3가지 용어를 파헤쳐 볼까요?

🔑 필수 키워드 3가지
  • ① 클래스 (Class)
    👉 "자바 프로그램의 가장 바깥 울타리입니다. 코드가 비를 맞지 않게 지어주는 아주 큰 건물이라고 생각하세요!"
  • ② 메인 메서드 (Main Method)
    👉 "건물(클래스)의 정문 출입구입니다. 자바는 public static void main(String[] args)라는 문이 없으면 프로그램 시작 자체를 못 합니다!"
  • ③ 주석 (Comment)
    👉 "컴퓨터는 읽지 못하고 사람만 볼 수 있는 투명 메모장이에요. // 표시 뒤에 자유롭게 설명을 적을 수 있어요."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

VS Code를 열고 HelloWorld.java라는 파일을 만들어 아래 코드를 똑같이 적어보세요. (대소문자 주의!)

// 1️⃣ 가장 큰 껍데기(클래스). 파일 이름과 반드시 똑같아야 합니다! (대문자로 시작)
public class HelloWorld {
    
    // 2️⃣ 자바 프로그램의 시작점 (메인 메서드 정문)
    public static void main(String[] args) {
        
        // 3️⃣ 컴퓨터에게 내리는 실제 명령어: 화면에 글자 출력 후 줄바꿈(ln)
        System.out.println("안녕하세요!");
        
        // 'ln'을 빼고 print만 쓰면 줄바꿈이 안 되고 옆으로 이어집니다.
        System.out.print("자바의 ");
        System.out.print("세계에 오신 것을 ");
        System.out.println("환영합니다!");
        
        // 숫자는 쌍따옴표 없이 바로 출력하고 계산도 할 수 있어요!
        System.out.println(100 + 200);
        
    } // 메인 메서드(방) 닫기
} // 클래스(건물) 닫기
[주의!] ⚠️ 초보자 흔한 실수와 해결책
  1. 🚨 빨간 줄이 떠요! ("세미콜론 누락"): 자바는 마침표를 찍지 않으면 문장이 안 끝났다고 생각합니다. println(); 끝에는 꼭 마침표인 세미콜론(;)을 찍으세요!
  2. 🚨 파일 내용과 파일 이름 불일치: public class HelloWorld 라고 지었다면 파일명은 HelloWorld.java 가 되어야 합니다. Helloworld.java (소문자 w)여도 에러가 납니다!
  3. 🚨 중괄호 짝꿍 미아: 여는 중괄호 { 가 있으면 반드시 닫는 중괄호 } 가 있어야 합니다. 내가 연 문을 잘 닫았는지 꼭 확인하세요!

[모듈 1: 기초 탄탄] 변수(Variable)와 서랍장 (기본 자료형)

1. 💡 눈높이 비유 (The Analogy)

💡 변수는 물건을 담는 **이름표 붙은 서랍장**입니다!

우리가 일상생활에서 물건을 정리할 때 어떻게 하나요?

  • 양말을 넣기 위한 칸에는 '양말 서랍'이라고 이름표(변수명)를 붙입니다.

문자와 문자열의 차이는 장난감 블록과 같습니다.

    👉 "여러 글자가 이어진 문장을 저장하는 큰 창고. 반드시 쌍따옴표(" ")를 사용해야 해요. (예: 이름 "홍길동", 주소 "서울시")"

  • **산술 연산자:** 기본 계산기 (사과를 3개씩 나눌 때 나머지는 몇 개일까?)

    👉 "나눗셈의 몫은 버리고 순수하게 '나머지'만 구해줍니다. 짝수/홀수를 판별할 때(2로 나눈 나머지) 마법처럼 쓰여요!"
  • ② 증감 연산자 ( ++, -- )
    👉 "score = score + 1; 이라고 길게 쓰기 귀찮죠? 그냥 score++; 하나면 현재 값에 1을 더해주는 프로그래머의 단축키입니다."
  • ③ 같음 비교 연산자 ( == )
    👉 "수학의 눈(=)과 다릅니다! 자바에서 은 = 대입(서랍에 넣기)이고, 양쪽이 진짜 똑같은지 비교할 땐 눈눈(==) 두 개를 써야 판사님이 'true/false' 판결을 내려줍니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

다양한 연산자 도구들을 실제 자바 코드에서 돌려봅시다!

public class OperatorTest {
    public static void main(String[] args) {
        
        // 1️⃣ 산술 연산자 (+, -, *, /, %)
        int a = 10;
        int b = 3;
        System.out.println("몫: " + (a / b));   // 10 나누기 3의 몫 = 3
        System.out.println("나머지: " + (a % b)); // 10 나누기 3의 나머지 = 1
        
        // 2️⃣ 증감 연산자 (++, --)
        int level = 1;
        level++; // 레벨 1 증가! (level = level + 1 과 완벽히 동일)
        System.out.println("현재 레벨: " + level); // 2 출력
        
        // 3️⃣ 비교 연산자 (>, <, >=, <=, ==, !=) -> 결과는 무조건 boolean(true/false)
        int myAge = 20;
        int yourAge = 20;
        
        boolean isSame = (myAge == yourAge); // 두 숫자가 완전히 똑같은가?
        boolean isAdult = (myAge >= 19);     // 19살보다 크거나 '같은가'?
        
        System.out.println("나이가 똑같나요? " + isSame); // true
        System.out.println("성인입니까? " + isAdult);   // true
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3

  2. 🚨 정수 나눗셈의 함정: 10 / 3을 하면 3.3333이 나올 것 같죠? 자바에서 정수(int)끼리 나누면 소수점은 자비 없이 버려버리고 몫인 3만 남습니다. 자세한 계산을 원하면 실수(double)로 바꿔야 해요! 10.0 / 3.0
  3. 🚨 문자열 비교는 `==` 금지: 나중에 배우겠지만 String 문장을 비교할 땐 == 저울을 쓰지 않고 .equals() 라는 특수한 마법 주문을 외워야 안전합니다!

지금까지의 컴퓨터는 위에서 아래로 무조건 한 뱡항으로만 달리는 직진 롤러코스터였습니다.

하지만 조건문(Condition)을 배우는 순간, 컴퓨터는 선택의 갈림길을 지닌 '내비게이션'으로 진화합니다!

  • **if (만약에 ~라면):** "만약 비가 온다면(조건), 우산을 챙긴다(행동)!"
  • **else if (그게 아니고, 또 다른 만약에):** "비는 안 오는데 눈이 온다면(조건2), 패딩을 챙긴다(행동2)!"
  • **else (위의 모든 경우가 아니라면):** "비도 안 오고 눈도 안 오면, 그냥 나간다!(기본 행동)"

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① if 구문
    👉 "괄호 ( ) 안의 내용이 참(true)일 때만 중괄호 { } 안방으로 들어가서 코드를 실행하는 철통 보안 경비원입니다."
  • ② else 구문
    👉 "if의 검문을 통과하지 못한(false) 모든 떨거지들을 다 받아주는 최후의 보루(마지막 도착지)입니다. 괄호로 조건식을 적지 않습니다!"
  • ③ 코드 블록 { }
    👉 "이 조건이 맞았을 때, 과연 어디서부터 어디까지 실행할 것인가? 그 범위를 지정해주는 울타리입니다."

public class ConditionIf {
    public static void main(String[] args) {
        
        int age = 15; // 내 나이
        
        // 1️⃣ 만약 나이가 19살 이상(성인) 이라면?
        if (age >= 19) {
            System.out.println("성인입니다. 삐빅- 1200원");
        }
        // 2️⃣ 성인이 아니고, 그게 아니라 만약 나이가 13살 이상(청소년) 이라면?
        else if (age >= 13) {
            System.out.println("청소년입니다. 삐빅- 720원"); // age가 15니까 이곳이 실행됩니다!
        }
        // 3️⃣ 위의 모든 조건이 아니라면? (어린이)
        else {
            System.out.println("어린이입니다. 삐빅- 450원");
        }
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3

  3. 🚨 순서의 반란: 위에서 아래로 물이 흐르듯 검사합니다! 만약 if(age>=13) 청소년 을 if(age>=19) 성인 보다 먼저 배치하면? 20살도 13이상이니 청소년으로 걸려서 통과해버리는 대형 사고가 납니다! (항상 좁고 구체적인 조건부터 먼저 쓰세요)

[모듈 1: 기초 탄탄] 조건문 2단계 (switch-case)

1. 💡 눈높이 비유 (The Analogy)

💡 자판기의 버튼 누르기!

앞서 배운 if문은 "만약 ~라면" 이라는 복잡한 조건을 여러 개 검사할 때 좋습니다. 하지만 switch문은 '정해진 여러 개의 버튼 중 하나를 딱 고르는 자판기'와 같습니다.

  • **switch (동전 넣기):** 내가 선택한 값(Value)을 기계에 넣습니다. (예: 메뉴 번호 2번!)
  • **case (버튼들):** 자판기에 달린 버튼들입니다. "만약 1번이 들어오면 이걸 줘, 2번이면 저걸 줘!" 하고 정해져 있죠.
  • **break (음료수 꺼내기):** 원하는 메뉴가 나왔으면 즉시 자판기에서 빠져나오는(정지) 브레이크입니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① switch (값)
    👉 "검사할 '정확한 값' 하나를 괄호 안에 던져줍니다. if문처럼 > 나 < 같은 범위 비교는 할 수 없어요!"
  • ② case 값:
    👉 "자판기의 각 버튼입니다. 던져진 값이 이 case의 값과 정확히 똑같을 때만 아래 명령을 실행합니다."
  • ③ break; 와 default:
    👉 "break는 실행을 멈추고 구문을 탈출하는 브레이크이고, default는 if문의 else처럼 '어떤 버튼에도 해당하지 않을 때' 나오는 기본값(에러 메시지 등)입니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

1, 2, 3번 메뉴가 있는 간단한 커피 자판기 코드를 만들어 봅시다.

public class ConditionSwitch {
    public static void main(String[] args) {
        
        int menuChoice = 2; // 사용자가 2번 버튼을 눌렀어요.
        
        // switch 자판기에 2라는 값을 던져 넣습니다!
        switch (menuChoice) {
            case 1:
                System.out.println("아메리카노가 나옵니다 ☕");
                break; // 1번이었으면 여기서 끝! 탈출!
                
            case 2:
                System.out.println("카페라떼가 나옵니다 🥛");
                break; // 값이 2니까 여기가 실행되고, 자판기를 빠져나옵니다.
                
            case 3:
                System.out.println("오렌지주스가 나옵니다 🍊");
                break;
                
            default:
                // 1, 2, 3이 아닌 엉뚱한 번호(예: 9번)를 눌렀을 때 실행됩니다. (if문의 else와 같음)
                System.out.println("없는 메뉴입니다. 돈을 반환합니다 💰");
                break;
        }
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 앗! 브레이크 고장 (break 누락): case 끝에 break;를 빼먹으면 어떻게 될까요? 2번을 눌러서 라떼가 나왔는데 시스템이 멈추지 않고 그 아래에 있는 오렌지주스까지 줄줄이 쏟아져 나오는 대참사(Fall-through 현상)가 발생합니다! 반드시 case마다 break를 달아주세요!
  2. 🚨 범위 지정 불가: case > 10: 이런 문법은 불가능합니다. switch-case는 오직 **'정확히 값이 일치하는가(==)'**만 검사할 수 있습니다.
  3. 🚨 콜론(:) 주의: case 1; 이 아니라 case 1: 입니다! 마침표(세미콜론)가 아니라 위아래 점 두 개인 콜론(:)임을 명심하세요.

[모듈 1: 기초 탄탄] 반복문 1단계 (for) - 정해진 횟수만큼 반복하기

1. 💡 눈높이 비유 (The Analogy)

💡 체육 선생님의 운동장 달리기 지시!

컴퓨터가 가장 잘하는 일은 '똑같은 일을 지치지 않고 수만 번 반복하는 것'입니다. for문은 체육 선생님이 학생들에게 내리는 명확한 지시와 같습니다.

  • **시작점:** "1번 학생부터 시작해!"
  • **도착점(조건):** "10번 학생이 될 때까지만 뛰어!"
  • **보폭(증감):** "한 번 뛸 때마다 번호를 1씩 올려가면서 뛰어!"

즉, "언제부터(초기화) ~ 언제까지(조건) ~ 얼만큼씩(증감)" 반복할지가 명확하게 정해져 있을 때 사용하는 최고의 도구입니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 초기화 (Initialization)
    👉 "루프(반복)를 돌리기 위한 카운트다운 타이머(변수)를 세팅합니다. 주로 int i = 0 또는 int i = 1 로 시작합니다. (i는 index의 약자)"
  • ② 조건식 (Condition)
    👉 "타이머가 어디까지 가야 멈출 것인지 정합니다. 예: i < 10 (i가 10보다 작을 때까지만 뺑뺑이를 돌아라!)"
  • ③ 증감식 (Increment/Decrement)
    👉 "한 바퀴 돌 때마다 타이머를 1씩 올릴지(i++) 내릴지(i--) 정합니다. 이거 안 쓰면 무한루프(영원히 안 끝남)에 빠져요!"

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

1부터 5까지 순서대로 세어주는 프로그램을 `for`문을 사용해 단 3줄로 작성해 봅시다.

public class LoopFor {
    public static void main(String[] args) {
        
        // for ( 1.초기세팅 ; 2.종료조건 ; 3.매바퀴증감 )
        for (int i = 1; i <= 5; i++) {
            
            // 이 괄호 안의 코드가 5번 반복 실행됩니다.
            System.out.println(i + "번째 달리기 중입니다!");
            
        } // 여기서 다시 위로 올라가서 i를 1 증가(i++)시키고 조건을 검사합니다.
        
        System.out.println("운동 끝!");
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 콤마(,)가 아니라 세미콜론(;): 괄호 안의 3가지 요소 사이에는 반드시 마침표 역할을 하는 세미콜론(;)이 들어가야 합니다. 쉼표(,)를 쓰면 에러가 납니다! for (int i=1, i<5, i++) // ❌틀림!
  2. 🚨 증감식에 세미콜론 찍기: 마지막 증감식 뒤에는 세미콜론을 찍지 않습니다. for (int i=1; i<=5; i++;) // ❌틀림!
  3. 🚨 i의 범위를 착각해요: i < 5 는 1, 2, 3, 4 까지만 실행됩니다. 숫자 5까지 꼭 포함하고 싶다면 작거나 같다인 <= 5를 쓰거나, < 6 으로 써야 한다는 점을 명심하세요!

[모듈 1: 기초 탄탄] 반복문 2단계 (while, break, continue)

1. 💡 눈높이 비유 (The Analogy)

💡 게임의 "살아있는 동안 계속 반복!" 규칙

앞서 배운 for 문이 '딱 10번만 뛰어!' 하고 횟수가 정해져 있을 때 쓴다면, while 문은 '언제 끝날지 정확히는 모르지만 이 조건이 유지되는 동안은 무조건 계속해!' 라고 할 때 사용합니다.

  • **while (조건):** "내 체력이 0보다 큰 동안에는(조건) 계속해서 몬스터를 때려라!(반복)"
  • **break (비상탈출):** "그러다 만약 전설의 보물을 발견하면? 하던 거 다 멈추고 즉시 탈출!"
  • **continue (스킵):** "돌을 주웠네? 이건 쓸모없으니 이번 턴만 패스하고 바로 다음 몬스터 찾아라!"

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① while (조건식)
    👉 "괄호 안의 조건이 참(true)인 동안 중괄호 안의 코드를 영원히 반복하는 햄스터 쳇바퀴입니다. 주로 while(true)로 무한루프를 만들 때 자주 써요."
  • ② break
    👉 "자판기(switch)에서 썼던 그 브레이크 맞습니다! 반복문 안에서 가장 가까운 쳇바퀴를 강제로 부수고 탈출하는 강력한 기능입니다."
  • ③ continue
    👉 "반복문을 완전히 탈출하진 않고, 밑에 남은 코드를 무시한 채 '다음 바퀴'로 점프(스킵)하게 만듭니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

1부터 차례대로 출력하다가, 특정한 조건에서 스킵(continue)하거나 멈추는(break) 코드를 보겠습니다.

public class LoopWhile {
    public static void main(String[] args) {
        
        int count = 1; // 카운트 변수 준비
        
        // 카운트가 10 이하인 '동안' 영원히 반복!
        while (count <= 10) {
            
            // 만약 카운트가 3이라면?
            if (count == 3) {
                count++; // 무한루프를 막기 위해 숫자를 하나 올려줍니다! (중요)
                System.out.println("숫자 3은 재수 없으니 패스!(continue)");
                continue; // 아래 코드를 안 하고 다음 바퀴로 점프!
            }
            
            // 만약 카운트가 5라면?
            if (count == 5) {
                System.out.println("숫자 5가 나왔다! 강제 탈출!(break)");
                break; // while문을 완전히 박살내고 탈출합니다.
            }
            
            // 정상 출력 파트
            System.out.println("현재 숫자: " + count);
            count++; // while문은 자동으로 숫자를 안 올려주니 우리가 직접 올려야 해요!
            
        } // -- while 닫기 --
        
        System.out.println("프로그램이 무사히 종료되었습니다.");
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 위험한 무한루프 (Infinite Loop): for문과 달리 while문은 괄호 안에 증감식(i++)이 없습니다! 쳇바퀴 안에서 숫자를 올려주지 않으면, count가 영원히 1에 머물러 컴퓨터가 멈출 때까지 글씨를 쏟아냅니다. 직접 증감식을 넣는 걸 절대 잊지 마세요!
  2. 🚨 continue 직전의 함정: if(count==3) 안에서 continue를 타고 바로 위로 올라가 버리면, 아래에 있던 count++를 만나지 못해 영원히 3인 상태로 무한루프에 빠집니다! 그래서 예제 코드에 continue 직전에 count++를 살짝 넣어둔 것입니다.
  3. 🚨 break 위치: break 밑에는 코드를 쓸 수 없습니다 (Unreachable code). 탈출해버렸으니 컴퓨터가 도달할 수 없거든요!

[모듈 2: 데이터 그룹화] 다차원 배열(2D Array) - 엑셀 표 같은 배열

1. 💡 눈높이 비유 (The Analogy)

💡 아파트 동호수나 엑셀 표 형태!

지난 시간에 배운 1차원 배열이 단층짜리 긴 복도 형태라면, 2차원 배열은 여러 층으로 이루어진 아파트(행과 열)입니다.

  • **행 (Row, 층수):** 가로줄을 말합니다. 아파트로 치면 1층, 2층, 3층... (물론 자바는 0층부터!)
  • **열 (Column, 호수):** 세로줄을 말합니다. 1호, 2호, 3호... (자바는 0호부터!)

예를 들어, 3명의 학생이 각각 국어, 영어, 수학 3과목의 점수를 가진다면, 이걸 1차원 배열로 9개 따로 만들기보단 3(학생) 줄 × 3(과목) 칸짜리 엑셀 표(2차원 배열)로 만드는 것이 훨씬 깔끔합니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 2차원 배열 선언 ( int[][] )
    👉 "대괄호 두 개를 연달아 붙이면 '행과 열을 가진 2D 표 모양이다!' 라고 알려주는 징표입니다."
  • ② 행과 열의 크기 할당 ( new int[행][열] )
    👉 "new int[3][4] 라고 쓰면 방이 3줄 있고, 각 줄마다 4칸씩 무언가를 넣을 수 있는 총 12칸짜리 아파트를 짓습니다."
  • ③ 중첩 for문 (Nested for Loop)
    👉 "2차원 배열의 모든 칸을 순서대로 열어보려면 for문 안에 또 for문을 넣는 '중첩 반복문'이 필수입니다. 바깥쪽 for문이 '층'을 내려갈 때, 안쪽 for문은 그 층의 '호수'를 쭉 훑습니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

2명 학생의 중간/기말고사 점수를 관리하는 2x2 표를 만들어 봅시다.

public class Array2D {
    public static void main(String[] args) {
        
        // 방법 1: 2행 2열짜리 표 만들고 나중에 값 채우기
        int[][] scores = new int[2][2];
        
        // 0번 학생 (행번호 0)
        scores[0][0] = 80; // 중간고사 점수
        scores[0][1] = 90; // 기말고사 점수
        
        // 1번 학생 (행번호 1)
        scores[1][0] = 70; // 중간고사 점수
        scores[1][1] = 85; // 기말고사 점수
        
        
        // 방법 2: 배열 선언과 동시에 중괄호로 표 그리기 (강력 추천!)
        int[][] gameGrid = {
            { 1, 2, 3 }, // 0번째 줄 (행)
            { 4, 5, 6 }  // 1번째 줄 (행)
        };
        
        System.out.println("gameGrid의 1행 2열(실제론 2번째줄 3번째칸) 값은? " + gameGrid[1][2]); // 6 출력
        
        // 중첩 for문으로 모든 엑셀 표 데이터 뽑아내기!
        System.out.println("\n[학생 점수 표 출력]");
        for (int row = 0; row < 2; row++) {      // 바깥 루프: 층(학생) 이동
            for (int col = 0; col < 2; col++) {  // 안쪽 루프: 호수(과목) 이동
                System.out.print(scores[row][col] + " "); // 한 줄에 이어서 찍기
            }
            System.out.println(); // 한 학생의 점수가 끝나면 줄바꿈
        }
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 행/열 순서 헷갈림: array[열][행] 처럼 거꾸로 쓰기 쉽습니다! 자바는 무조건 array[행(Y, 세로축)][열(X, 가로축)] 순서입니다. 엑셀을 켤 때 '몇 번째 줄(행)'에 있는 '몇 번째 칸(열)'을 고른다고 생각하세요.
  2. 🚨 array.length 의 의미 파악: 그냥 array.length라고 치면 줄의 개수(행)가 나옵니다. 각 줄에 들어있는 칸의 개수(열)를 알고 싶다면 array[0].length 처럼 행을 먼저 찍고 물어봐야 합니다.
  3. 🚨 out of bounds 주의: 1차원에서도 그랬듯 방 번호 잘못 부르면 바로 프로그램 터집니다! new int[2][3] 라면 마지막 방은 [1][2] 입니다.

1. 💡 눈높이 비유 (The Analogy)

💡 무한히 늘어나는 해리포터의 마법 가방!

배열(Array)은 참 좋지만, 한 가지 치명적인 단점이 있었습니다. 처음에 "방 5개!" 하고 만들어버리면, 손님이 6명이 오더라도 방을 늘려줄 수가 없었죠(크기 고정).

그래서 자바가 발명한 것이 바로 ArrayList(마법의 가방) 입니다!

  • 이 가방은 물건을 쑤셔 넣으면 넣을수록 내부 공간이 신축성 있게 알아서 쭈욱쭈욱 늘어납니다.
  • 반대로 물건을 빼서 버리면, 빈 공간이 허전하지 않게 가방 크기가 스스로 착! 줄어듭니다.
  • 현업에서는 고정된 크기의 배열보다, 이렇게 자유자재로 변하는 ArrayList를 10배는 더 많이 사용합니다!

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 선언 제네릭 ( <타입> )
    👉 "이 가방엔 아무거나 막 넣는 게 아니라, ArrayList<String> 처럼 '오직 문자열 장비만 담는 가방!' 이라고 미리 성격을 명확히 정해줍니다. (이 꺾쇠 기호를 '제네릭'이라고 불러요)"
  • ② 추가와 삭제 ( .add() , .remove() )
    👉 "배열의 [번호] = 값 대신, 가방이 제공하는 전용 지퍼 기능을 씁니다. add("칼") 하면 맨 뒤에 장비가 쏙 들어가고, remove(0) 하면 0번째 장비가 삭제되며 빈자리를 자동으로 스르륵 당겨 메워줍니다!"
  • ③ 크기 확인 ( .size() )
    👉 "배열은 .length로 길이를 재지만, 마법 가방은 현재 가방 안에 물건이 몇 개나 들었는지 .size() 포켓몬 도감 같은 기능으로 개수를 세어줍니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

// 주의: ArrayList를 쓰려면 꼭 맨 위에 수입(import) 해와야 쓸 수 있습니다!
import java.util.ArrayList;

public class MagicBagTest {
    public static void main(String[] args) {
        
        // 1️⃣ 가방 생성 (String 아이템만 들어가는 빈 가방 메기)
        ArrayList<String> inventory = new ArrayList<>();
        
        // 2️⃣ 아이템 추가 (add) - 넣는 순서대로 차곡차곡 쌓임! 0번부터~
        inventory.add("낡은 목검");  // 0번 방
        inventory.add("빨간 포션");  // 1번 방
        inventory.add("가죽 갑옷");  // 2번 방
        
        // 특정 번호표 구역에 물건 끼워넣기 (새치기!)
        inventory.add(1, "성스러운 방패"); // 1번 자리에 방패를 억지로 끼워넣음 (포션은 2번으로 밀려남)

        System.out.println("물건 개수: " + inventory.size() + "개");

        // 0번(낡은 목검)을 버리면, 나머지 물건들이 빈자리를 메우기 위해 앞으로 딸려옵니다!

        System.out.println("지금 0번 방에 있는 건? : " + inventory.get(0)); // 목검이 사라지고 방패가 딸려왔을 거에요!
        
    }
}

  3. 🚨 배열과 헷갈린 문법: 조회할 때 inventory[0] 라고 대괄호를 쓰면 에러입니다! 가방은 문을 열어달라고 하는 .get(0) 이라는 전용 요청 버튼을 써야 합니다.

[모듈 2: 데이터 그룹화] HashMap 기초 - 이름표가 붙은 마법의 주머니

1. 💡 눈높이 비유 (The Analogy)

💡 목욕탕 아주머니의 열쇠와 옷 바구니!

배열이나 ArrayList는 물건을 보관할 때 방 번호(0, 1, 2...)만 기억했습니다. "125번째 서랍에 여권이 들어있어!" 라고 외우려니 사람이 쓰기엔 너무 헷갈립니다.

HashMap은 '단어가 적힌 이름표(Key)'와 '실제 물건(Value)'이 세트로 들어가는 스마트 주머니입니다.

  • **Key (열쇠/이름표):** 오직 한 개만 존재하는 고유한 이름표입니다. (예: "내여권", "엄마번호")
  • **Value (물건/데이터):** 그 이름표에 줄로 묶여 딸려오는 실제 데이터입니다. (예: 여권실물, 010-1234-5678)
  • 이제 방 번호는 잊으세요! "지갑 꺼내줘!" 하면 지갑이 나오고, "현관비밀번호 꺼내줘!" 하면 비밀번호가 나옵니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① Key-Value 한 쌍 ( <K, V> )
    👉 "생성할 때 HashMap<String, Integer> 처럼 두 가지를 정합니다. 앞은 '이름표의 재질(문자열)'이고 뒤는 '내용물의 종류(숫자)'입니다. (예: 이름표 '국어' ➔ 점수 '90')"
  • ② 넣기 ( .put() )
    👉 "ArrayList는 무식하게 .add()로 던져 넣었지만, HashMap은 .put("이름표", 물건) 처럼 항상 이름표와 물건을 같이 제출해야 주머니에 들어갑니다."
  • ③ 꺼내기 ( .get() )
    👉 "꺼낼 때는 방 번호가 아니라 .get("이름표") 처럼 열쇠(Key)를 제시해야만 보관해둔 물건을 받을 수 있습니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

친구들의 나이를 헷갈리지 않게 '이름표-나이' 세트로 저장하는 연락처 앱을 만들어 볼까요?

// 사용하려면 반드시 import 필요!
import java.util.HashMap;

public class MapTest {
    public static void main(String[] args) {
        
        // 1️⃣ HashMap 생성 (이름표: String 문자, 내용물: Integer 숫자)
        HashMap<String, Integer> ages = new HashMap<>();
        
        // 2️⃣ 데이터 넣기 (put) - <Key, Value> 세트
        ages.put("철수", 20);
        ages.put("영희", 22);
        ages.put("민수", 21);
        
        // 3️⃣ 데이터 꺼내보기 (get) - 방 번호(0)가 아닌 이름표(Key)로 꺼냄!
        System.out.println("영희의 나이는? " + ages.get("영희") + "살");
        
        // 4️⃣ 데이터 변경하기 (같은 Key에 put을 또 하면 덮어쓰기 됨)
        // 영희가 한 살 지났다면? "영희"라는 열쇠로 새 나이 23을 넣습니다!
        ages.put("영희", 23);
        System.out.println("해를 넘긴 영희 나이: " + ages.get("영희") + "살");
        
        // 5️⃣ 데이터 지우기 (remove) - 키 값만 주면 가방에서 꺼내서 버립니다.
        ages.remove("민수"); 
        System.out.println("민수가 이민을 갔습니다. 현재 연락처: " + ages);
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 앗! 중복 열쇠 (Key 덮어쓰기): ArrayList는 "사과"를 여러 번 넣어도 각각 저장되지만, HashMap의 Key는 절대 중복될 수 없습니다. 이미 "영희"가 있는데 .put("영희", 99)를 해버리면 기존 영희의 데이터는 삭제되고 99로 무자비하게 덮어쓰여집니다!
  2. 🚨 없는 열쇠로 열기 (null 반환): ages.get("동건") 처럼 주머니에 넣지도 않은 이름표를 달라고 하면, 에러는 나지 않지만 텅 비었다는 뜻인 null 이라는 유령 값을 던져줍니다. 나중에 계산하려다 터질 수 있으니 조심하세요!
  3. 🚨 순서의 배신: 내가 "철수", "영희", "민수" 순으로 다이어리에 적어 넣어도, System.out.println(ages) 로 확인해보면 컴퓨터 마음대로 마구 뒤섞여 저장되어 있습니다! HashMap은 속도를 위해 바구니 안에 아무렇게나 짐을 던져놓는 구조(순서 보장 안 됨)이기 때문입니다.

1. 💡 눈높이 비유 (The Analogy)

💡 붕어빵을 끝없이 구워내는 마법의 틀!

지금까지는 int, String 같은 기본 재료만으로 프로그램을 짰습니다. 하지만 복잡한 시스템(예: 은행, 게임)을 만들려면 나만의 복합적인 재료 세트가 필요합니다.

이것이 바로 객체지향 프로그래밍(OOP)의 시작, 클래스와 객체입니다!

  • **클래스(Class): 붕어빵 틀 (설계도)**
    "머리부터 꼬리까지 이런 모양이고, 안에는 팥이 들어간다!"라고 그려놓은 도면이나 거푸집입니다. 눈으로 볼 수 있는 실체는 아니고, 단지 '설명서'일 뿐입니다.
  • **객체(Object / Instance): 만들어진 붕어빵 (실체)**
    설계도를 바탕으로 밀가루를 부어 구워낸 실제 먹을 수 있는 붕어빵입니다. 팥 붕어빵 한 개, 슈크림 붕어빵 한 개, 치즈 붕어빵 한 개... 이 도면(Class) 하나만 있으면 실체(Object)를 100개든 1000개든 무한으로 찍어낼 수 있습니다!

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 클래스 선언 ( class 이름 {} )
    👉 "class Dog 라고 적으면, '이제부터 컴퓨터 안에 나만의 반려견을 만드는 틀을 정의하겠다!' 라는 선언입니다."
  • ② 인스턴스화 ( new 클래스명() )
    👉 "붕어빵 틀에서 구워내는 행동을 '인스턴스화'라고 합니다. new Dog() 라고 마법의 주문을 외치면, 메모리라는 쟁반 위에 강아지 형체 1마리가 뿅! 하고 진짜로 생겨납니다."
  • ③ 객체 참조 변수 ( 클래스명 변수명 = new ... )
    👉 "생성된 강아지는 이리저리 도망갈 수 있습니다. 그래서 Dog myMax = new Dog() 처럼, myMax라는 이름의 목줄(참조 변수)에 묶어두어야 우리가 마음대로 부를 수 있습니다."

// 1️⃣ 자동차(Car) 설계도(클래스)를 만듭니다. (실제 차가 아님!)
class Car {

    String modelName; // 모델명
    String color;     // 색상
    
    // 자동차가 할 수 있는 행동 (메서드)
    void honk() {
        System.out.println(modelName + " : 빵빵! 비키세요!");
    }

// 프로그램을 실행하는 본부석 (Main 클래스)
public class FactoryMain {

        Car car1 = new Car(); 
        car1.modelName = "소나타"; // 점(.)을 찍어서 속성에 접근
        car1.color = "검정색";
        
        // 3️⃣ 두 번째 자동차 조립(생성)! - 틀은 하나지만 여러 개 무한 생산 가능
        Car car2 = new Car(); 
        car2.modelName = "모닝";
        car2.color = "노란색";
        
        // 4️⃣ 각 자동차에게 행동 명령 내리기
        car1.honk(); // 소나타 : 빵빵! 비키세요!
        car2.honk(); // 모닝 : 빵빵! 비키세요!
        
    }
}

[모듈 3: 클래스와 객체] 속성(필드)과 동작(메서드) - 객체에 데이터와 기능 넣기

1. 💡 눈높이 비유 (The Analogy)

💡 게임 캐릭터 분석하기! (명사와 동사)

우리가 게임에서 마법사 캐릭터를 만들었다고 상상해 봅시다. 이 마법사는 혼자서 완성되지 않습니다. 이 캐릭터를 구성하는 두 가지 중요한 기둥이 있습니다.

  • **필드(Field) = 명사 / 상태 / 스펙**
    마법사의 이름, 체력(HP), 레벨, 남은 마나(MP) 같은 것들입니다. 캐릭터가 "가지고 있는 정보" 혹은 "상태값"을 뜻합니다.
  • **메서드(Method) = 동사 / 동작 / 기능**
    마법사가 달린다(), 파이어볼을 쏜다(), 휴식하여 체력을 회복한다() 같은 것들입니다. 캐릭터가 스스로 할 수 있는 "능력"이나 "행동"을 뜻합니다.

객체(캐릭터)는 이처럼 필드(정보)와 메서드(행동)가 하나로 찰떡처럼 뭉쳐진 완전체입니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 필드 선언 ( 전역 변수 )
    👉 "클래스의 중괄호 {} 바로 안쪽에 선언된 변수를 '필드' 또는 '멤버 변수'라고 합니다. 잠깐 쓰다 버리는 지역변수와 달리, 객체가 살아있는 동안 함께 영원히 유지되는 소중한 데이터입니다."
  • ② 메서드 선언 ( 리턴타입 이름() {} )
    👉 "객체가 할 수 있는 행동을 정의합니다. void attack() { ... } 처럼 만들어 두면, 나중에 캐릭터.attack(); 한 번으로 공격 기능을 재사용할 수 있습니다."
  • ③ 도트 연산자 ( . )
    👉 "만들어진 객체에게 접근하는 만능 리모컨 버튼입니다! hero.hp = 100; (필드 정보 건드리기), hero.run(); (메서드 행동 시키기) 처럼 이름 뒤에 점(.)을 찍어 모든 걸 조종합니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

전사 캐릭터의 설계도를 꼼꼼히 짜고, 그 안에서 필드와 메서드가 어떻게 연동되는지 봅시다.

// 1️⃣ 설계도(클래스) 영역
class Warrior {
    // --- [필드: 상태/명사] ---
    String name;  // 캐릭터 이름
    int hp = 100; // 체력 (기본값 100을 줘서 태어남)
    int level = 1; // 레벨
    
    // --- [메서드: 행동/동사] ---
    // 1. 공격 기능
    void attack() {
        System.out.println(name + "이(가) 호쾌하게 칼을 휘두릅니다! 슉!");
    }
    
    // 2. 피해받기 기능 (매개변수를 통해 데미지를 받음!)
    void takeDamage(int damage) {
        hp = hp - damage; // 내 체력(필드)에서 들어온 데미지만큼 깎음
        System.out.println(name + "이(가) " + damage + "의 피해를 입었습니다! (남은 체력: " + hp + ")");
    }
}

// 2️⃣ 실행(본부석) 영역
public class GameMain {
    public static void main(String[] args) {
        
        // 전사 객체 1 생성!
        Warrior knight = new Warrior();
        
        // 점(.)을 찍어 필드에 접근하여 이름표 달아주기
        knight.name = "아서왕"; 
        
        // 점(.)을 찍어 메서드(행동) 호출하기
        knight.attack(); 
        
        // 몬스터에게 30의 데미지를 받았다고 지시 내리기
        knight.takeDamage(30); 
        
    }
}

  3. 🚨 초기화 없이 필드 사용: 숫자는 0, 문자열은 null이 기본값으로 들어갑니다. 이름을 지어주지도 않고 attack()을 부르면 "null이(가) 휘두릅니다!" 라며 바보 같은 출력이 나오니 객체를 만든 후 생명을 불어넣어(초기화) 주세요.

[모듈 3: 클래스와 객체] 생성자(Constructor) - 태어날 때 꼭 입어야 할 교복

1. 💡 눈높이 비유 (The Analogy)

💡 아기 출생신고서와 병원 팔찌!

지금까지는 객체를 먼저 만들고(new Car()), 나중에 따로 접근해서 이름을 지어줬습니다(car.name = "모닝"). 이 방식은 가끔 실수를 낳습니다. 깜빡하고 이름을 안 지어주면 '이름 없는 유령 객체'가 되거든요.

생성자(Constructor)는 객체가 태어나는(new) 바로 그 순간, 강제로 이름을 짓거나 필수 작업(초기화)을 하게 만드는 멱살 잡기 도구입니다.

  • 병원에서 아기가 태어나자마자 이름표 팔찌를 채웁니다. 팔찌 없이는 신생아실을 나갈 수 없는 규칙을 만드는 거죠!
  • "너 자동차 하나 만들 거면, 처음부터 '색상'이랑 '모델명'은 꼭 제출해! 안 그러면 안 만들어줄 거야!" 라고 강제하는 보험 같은 존재입니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 생성자의 이름은 무조건 클래스명과 동일!
    👉 "클래스가 Car라면, 생성자 이름도 Car() 여야만 자바가 '아, 얘가 이 객체를 태어나게 하는 담당의사구나!' 하고 인식합니다. 소문자 car()로 쓰면 그냥 일반 메서드가 되어버립니다."
  • ② 리턴 타입(void 등)이 아예 없음
    👉 "일반 메서드는 void run(), int getAge() 처럼 타입이 붙지만, 생성자는 리턴 타입 자체가 존재하지 않습니다. 오직 public Car() { ... } 모양입니다."
  • ③ 기본 생성자 (Default Constructor)
    👉 "우리가 아무 생성자도 직접 안 만들면, 자바가 뒤에서 몰래 텅 빈 생성자(public Car(){})를 줘서 조용히 태어나게 해줍니다. 하지만 우리가 하나라도 깐깐한 조건의 생성자를 직접 만들면, 이 공짜 혜택은 수거해 갑니다!"

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

메뉴 이름과 샷 횟수가 꼭 적혀야만 만들어지는 강력한 '커피(Coffee)' 클래스를 확인해 봅시다.

class Coffee {
    String name;  // 커피 이름
    int shots;    // 샷 횟수
    
    // 🚨 [생성자] : Coffee 객체가 new로 태어날 때 '무조건' 한 번만 실행됨!
    // 밖에서 이름을 던져주면, 내 필드에 찰칵! 조립해줍니다.
    Coffee(String inputName, int inputShots) {
        name = inputName; // 넘겨받은 이름을 내 필드로 세팅
        shots = inputShots;
        
        System.out.println(name + " 주문이 들어왔습니다. (초기화 완료!)");
    }
    
    void serve() {
        System.out.println(name + "(샷 " + shots + "번) 한 잔 나왔습니다 ☕");
    }
}

public class CafeMain {
    public static void main(String[] args) {
        
        // ❌ 에러 발생 코드: Coffee c1 = new Coffee(); 
        // 이유: 우리가 이름과 샷을 무조건 받으라고 깐깐한 생성자를 만들었기 때문에 공짜 입장 불가!
        
        // ✅ 이렇게, 생성자 괄호 안에 태어나자마자 들어갈 재료들을 쏴주어야 합니다.
        Coffee c2 = new Coffee("아메리카노", 2); 
        Coffee c3 = new Coffee("라떼", 1);

[모듈 3: 클래스와 객체] 캡슐화(Encapsulation) - 은행 금고와 ATM 기계

1. 💡 눈높이 비유 (The Analogy)

💡 은행 금고를 숨기고, ATM만 열어두자!

만약 은행 계좌 클래스의 잔액(balance) 접근이 누구에게나 허용된다면 어떨까요?
해커(다른 클래스)가 myAccount.balance = 99999999; 라고 자기 마음대로 잔고를 조작해버릴 수 있습니다. 상상만 해도 끔찍하죠?

캡슐화(Encapsulation)는 이렇게 민감한 데이터(필드)를 아무도 못 건드리게 꽁꽁 숨기고(private), 오직 철저한 검사를 거친 안전한 창구(Getter/Setter 메서드)를 통해서만 접근하도록 통제하는 기술입니다.

  • **private 필드 = 철창이 쳐진 은행 금고:** 오직 그 은행(클래스 내부)에서만 열 수 있습니다. 외부 접근 절대 차단!
  • **public Getter/Setter = 은행 창구 직원 (ATM):** 고객이 비밀번호를 제대로 쳤는지, 마이너스 금액을 입금하진 않는지 깐깐하게 검사한 뒤 통과시켜 줍니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 접근 제어자 private
    👉 "자물쇠입니다! 변수 이름 앞에 private int age; 라고 달아두면, 나(현재 클래스) 말고 다른 어떤 누구도 이 변수에 접근하거나 값을 수정할 수 없게 철벽 방어를 합니다."
  • ② Setter (세터 - 값을 저장/수정하는 대리인)
    👉 "외부에서 금고(필드)에 값을 넣고 싶을 때 부르는 직원의 이름(관례)입니다. 보통 set변수명() 모양을 가집니다. 여기서 '나이가 0살 미만인가요?' 같은 유효성 검사를 진행합니다."
  • ③ Getter (게터 - 값을 읽어오는 대리인)
    👉 "외부에서 금고(필드)의 값을 구경하고 싶을 때 부르는 직원의 이름입니다. get변수명() 모양을 가지며, 안전하게 복사본만 보여주고 진짜 데이터는 지켜냅니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

나이를 절대로 음수(-)로 설정할 수 없는 완벽한 사람(Person) 클래스를 만들어 봅시다!

class Person {
    // 1️⃣ 필드를 무조건 private으로 '은닉'합니다! 외부 접근 차단.
    private String name;
    private int age;
    
    // 2️⃣ 이름 Setter / Getter
    public void setName(String n) {
        name = n;
    }
    public String getName() {
        return name;
    }
    
    // 3️⃣ 나이 Setter (문지기 역할 톡톡히 하기!)
    public void setAge(int a) {
        // 무조건 넣는 게 아니라 거름망(조사)을 거칩니다.
        if (a < 0) {
            System.out.println("🚨 경고: 나이는 0보다 작을 수 없습니다! (기본값 0 설정)");
            age = 0;
        } else {
            age = a;
        }
    }
    
    // 나이 Getter
    public int getAge() {
        return age;
    }
}

public class EncapsulationMain {
    public static void main(String[] args) {
        
        Person p = new Person();
        
        // ❌ 에러 발생: p.age = -50; 
        // (private이라 안 보여서 조작 불가! 해킹 철벽 방어!)
        
        // ✅ 오직 허락된 창구(Setter)를 통해서만 값을 넣습니다.
        p.setName("유재석");
        p.setAge(-50); // 문지기가 막아내고 0으로 세팅해줍니다.
        
        // 값을 꺼내 볼 때도 허락된 창구(Getter)를 통해서!
        System.out.println(p.getName() + "님의 나이는 " + p.getAge() + "살 입니다.");
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 귀찮다고 전부 public으로 열어두기: 공부할 땐 편하지만 실무에서 그렇게 짰다간 사수한테 엄청나게 혼납니다. 객체지향의 기본 철칙은 "필드(변수)는 무조건 private, 메서드는 보통 public" 입니다!
  2. 🚨 Getter에서 값 반환 안 해주기: getAge() 메서드는 내 나이가 몇 살인지 물어보는 창구입니다. 따라서 반드시 int getAge() {...} 처럼 리턴 타입을 적고, 끝에 return age; 를 써서 값을 돌려보내 줘야 합니다.
  3. 🚨 IDE(VS Code)의 자동 완성 꿀팁: 현업에선 Getter/Setter를 일일이 수작업으로 치지 않습니다! VS Code나 이클립스에서 필드만 만들어 놓고 우클릭(또는 Quick Fix) 메뉴를 보면 "Generate Getters and Setters"라는 마법의 버튼이 다 만들어 줍니다.

[모듈 4: 객체지향 심화] 상속(Inheritance) - 부모의 유산을 그대로 물려받기

1. 💡 눈높이 비유 (The Analogy)

💡 백종원의 만능 간장 베이스!

요리를 할 때 떡볶이, 제육볶음, 찜닭을 처음부터 끝까지 다 따로 만들면 양념(코드)을 3번 만들어야 해서 너무 귀찮습니다.

객체지향 프로그래밍의 핵사이다, 상속(Inheritance)은 마치 '백종원표 만능 간장(부모 클래스)'을 만들어 두는 것과 같습니다.

  • 만능 간장 베이스(부모)를 만들어 두고, 여기서 확장(상속)하여 파만 넣으면 '찜닭 소스(자식 1)'가 되고, 고추장을 추가하면 '제육 소스(자식 2)'가 됩니다.
  • 기본 메뉴얼(코드)을 복사/붙여넣기 할 필요 없이, 부모의 모든 재산(필드)과 기술(메서드)을 그냥 꿀꺽! 편하게 얻어오고 내 것만 덧붙이면 끝입니다. 중복 코드가 확 줄어들죠!

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 상속 키워드 extends (확장하다)
    👉 "자식 클래스 이름 뒤에 extends 부모클래스 라고 적습니다. 자바에서는 상속을 '받는다'기보단, 부모를 발판 삼아 내 능력을 더 '확장(extends)'한다는 아주 긍정적인 의미로 씁니다."
  • ② 부모/슈퍼 클래스 (Super Class)
    👉 "유산을 물려주는 쪽입니다. (예: 스마트폰, 동물, 자동차 같은 큼직큼직하고 공통적인 뼈대)"
  • ③ 자식/서브 클래스 (Sub Class)
    👉 "유산을 받는 쪽입니다. 부모의 능력을 기본 패시브로 다 쓰면서 자기만의 특수 능력도 가집니다. (예: 아이폰, 고양이, 스포츠카 같은 세부적인 실체)"

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

기본 스마트폰(부모)의 능력을 물려받고 애플페이 특수능력을 추가한 아이폰(자식)을 만들어 봅시다.

// 1️⃣ 만능 베이스! 부모 클래스: 스마트폰
class Phone {
    String model;    // 기종
    String number;   // 전화번호
    
    // 부모의 공통 기술 (물려줄 유산)
    void call() {
        System.out.println(number + " 번호로 전화를 겁니다. 따르릉~");
    }
}

// 2️⃣ 자식 클래스: 아이폰 (Phone의 모든 능력을 '확장'해서 가져옴!)
class IPhone extends Phone {
    
    // ⚠️ 주목! 자식 클래스 안에는 call(), model 같은 걸 또 적을 필요가 전혀 없습니다! 이미 내 몸 안에 다 있습니다.
    
    // 오직 나만의 특수 능력(알파)만 추가합니다.
    void applePay() {
        System.out.println(model + " 모델로 애플페이 띠링! 결제가 완료되었습니다.");
    }
}

// 본부석 (실행)
public class ExtendMain {
    public static void main(String[] args) {
        
        // 아이폰 하나 샀습니다.
        IPhone myPhone = new IPhone();
        
        // 1. 부모가 물려준 변수 사용 가능!
        myPhone.model = "아이폰 15";
        myPhone.number = "010-1111-2222";
        
        // 2. 부모가 물려준 통화 기술 사용 가능!
        myPhone.call(); 
        
        // 3. 내 고유 기술인 애플페이 사용 가능!
        myPhone.applePay(); 
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 다중 상속 불가능: 엄마의 능력도 받고 싶고 아빠의 능력도 받고 싶다며 extends Mom, Dad 처럼 두 부모를 동시에 적으면 자바는 즉시 에러를 뿜어냅니다! 자바에서 부모 클래스는 무조건 단 1개만 지목할 수 있습니다.
  2. 🚨 private 유산은 물려줄 수 없음: 부모 거북이가 private int secretGold = 1000; 이라고 꽁꽁 숨겨둔 무덤 속 비자금은 자식이라도 절대 접근 불가입니다. 물려주고 싶다면 public이나 protected를 써야 합니다.
  3. 🚨 의미 없는 남용 주의: 그랜저에다가 억지로 냉장고를 `extends` 시키면 안 됩니다! 상속은 반드시 "A는 B이다(A is a B)" 관계가 논리적으로 성립할 때만 써야 합니다. (예: "아이폰은 스마트폰이다." ➔ OK!)

[모듈 4: 객체지향 심화] 다형성(Polymorphism)과 오버라이딩 - 내 입맛대로 부모 능력 바꾸기!

1. 💡 눈높이 비유 (The Analogy)

💡 카멜레온의 변신술과 "아빠, 난 좀 다르게 할래!"

다형성(Polymorphism)이란 '하나의 객체가 여러 가지 형태를 가질 수 있는 성질'을 뜻합니다. 어려어 보이만 실생활에서 흔히 쓰입니다.

  • **다형성:** 동물병원 의사가 "다음 동물(Animal) 들어오세요~ 진찰할게요!" 라고 했을 때, 강아지가 들어와도 진찰() 가능하고, 고양이가 들어와도 진찰()이 가능합니다. 모두 '동물'이라는 부모 이름표로 묶어 부를 수 있기 때문입니다.
  • **오버라이딩(Overriding):** 부모(Animal)가 물려준 울음소리내기() 기술을 물려받았는데, 강아지(자식)는 "난 멍멍이라고 할래!", 고양이(자식)는 "난 야옹이라고 덮어쓸래!" 라며 부모의 기술(메서드)을 자기 입맛대로 뜯어 고치는(덮어쓰기) 것입니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 오버라이딩 (Overriding = 덮어쓰기)
    👉 "부모 클래스에 이미 있는 메서드를, 자식 클래스에서 똑같은 이름, 똑같은 괄호() 형태로 다시 적어서 내용물만 교체하는 겁니다."
  • ② @Override 어노테이션
    👉 "메서드 바로 위에 달아두는 이름표입니다. 이거 부모 메서드 덮어쓴 거예요! 라고 컴퓨터(컴파일러)에게 알려줘서, 혹시 내가 오타를 냈다면 바로 에러를 띄워 짚어주는 고마운 보호막입니다."
  • ③ 다형적 형변환 ( 부모타입 변수 = new 자식() )
    👉 "Animal a = new Dog(); 라고 쓸 수 있는 마법입니다. 개(Dog)는 일종의 동물(Animal)이므로, 넓은 의미의 동물 바구니에 개를 담아둘 수 있습니다!"

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

모든 동물(부모)은 소리를 낼 수 있습니다. 하지만 이 유산을 각 동물(자식)들끼리 어떻게 다르게 사용하는지 볼까요?

// 1️⃣ 부모 클래스: 동물 베이스
class Animal {
    // 보편적인 동물의 울음소리 (유산)
    void makeSound() {
        System.out.println("...동물이 소리를 냅니다...");
    }
}

// 2️⃣ 자식 클래스: 개 (부모의 소리를 멍멍!으로 조작)
class Dog extends Animal {
    // 부모의 makeSound()를 내 방식대로 덮어씁니다!
    @Override 
    void makeSound() {
        System.out.println("왈왈!! 멍멍!!");
    }
}

// 3️⃣ 자식 클래스: 고양이 (부모의 소리를 야옹!으로 조작)
class Cat extends Animal {
    @Override 
    void makeSound() {
        System.out.println("야~~~옹 냐아아앙");
    }
}

// 본부석
public class PolyMain {
    public static void main(String[] args) {
        
        // --- [다형성의 마법] ---
        // 부모 이름표(Animal) 하나로 여러 자식들을 일렬로 세울 수 있습니다!
        Animal myDog = new Dog(); 
        Animal myCat = new Cat();
        
        // 똑같이 "소리내봐!" (makeSound) 라고 불렀는데, 대답이 각기 다릅니다!
        myDog.makeSound(); // 출력: 왈왈!! 멍멍!!
        myCat.makeSound(); // 출력: 야~~~옹 냐아아앙
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 오버라이딩 vs 오버로딩 혼동:
    - 오버라이딩(Overriding): 부모 클래스에 올라타서 부모의 것을 덮어쓰기 (서로 다른 클래스 간)
    - 오버로딩(Overloading): 내 클래스 안에서 여러 개체를 담기 위해 매개변수만 바꿔서 다양하게 쌓아 올리기 (같은 클래스 안) 이 두 개념은 완전히 다릅니다!
  2. 🚨 메서드 이름 오타: 부모는 makeSound()인데 자식이 실수로 makesound()(소문자 s)로 만들면, 자바는 덮어쓰기가 아니라 '아, 새로운 메서드 만들었구나' 하고 전혀 다른 취급을 합니다. @Override 어노테이션을 쓰면 이런 오타를 막아줍니다!
  3. 🚨 반대로 담기: Animal a = new Dog()는 가능하지만, Dog d = new Animal()은 불가능합니다! "모든 강아지는 동물이지만, 모든 동물이 다 강아지는 아니다" 라는 논리를 꼭 기억하세요.

[모듈 4: 객체지향 심화] 인터페이스(Interface) - 약속과 규격 정하기 (리모컨의 비밀)

1. 💡 눈높이 비유 (The Analogy)

💡 컴퓨터의 USB 포트, 그리고 리모컨 규격!

우리가 삼성전자, LG전자, 다이소 등 어떤 브랜드의 마우스를 사더라도 그냥 'USB 단자 구멍'에 꽂으면 작동합니다. 왜 그럴까요?
전 세계 모든 마우스 회사들이 "USB라는 규격(약속)을 무조건 지키자!" 라고 서약서를 썼기 때문입니다.

인터페이스(Interface)가 바로 자바의 전 세계 공통 '규격'이자 철저한 '약속'입니다.

  • 추상적인 빈 껍데기 버튼(규격)만 만들어둡니다. (예: "모든 리모컨은 무조건 전원켜기 버튼을 가져야 해!")
  • 이 약속에 도장을 찍은(implements) 사람들은, 강제로 그 버튼을 어떻게 작동시킬지 각자 알아서 완벽하게 구현해야 합니다. (협업에 필수!)

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① interface 선언
    👉 "class 대신 interface 라고 적습니다. 이 안에는 몸통(중괄호 {}) 없는 텅 빈 메서드(추상 메서드)들만 가득 나열해서 규격만 적립합니다."
  • ② implements (구현하다)
    👉 상속이 extends 였다면, 인터페이스는 implements를 씁니다. "나 클래스 A는, 이 인터페이스 규격을 완벽하게 내 몸속에 구현해서 작동시키겠다!" 라는 강제 서약서입니다.
  • ③ 다중 구현 허용!
    👉 "부모(상속)는 오직 1명만 모실 수 있지만, 인터페이스는 규격일 뿐! 그래서 '나는 USB 규격도 따르고(implements), 블루투스 규격도 따를래!' 라며 implements A, B, C 처럼 무한대로 약속에 가입할 수 있습니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

'비행 기능(Flyable)' 이라는 자격증(약속) 규격을 만들고, 비행기와 새가 이를 어떻게 구현하는지 봅시다.

// 1️⃣ 규격 설계도 (인터페이스)
// "이 규격에 가입하고 싶으면, 반드시 날아오르는 fly() 기술을 만들어야 해!"
interface Flyable {
    // 껍데기만 존재함! { 블록 } 이 절대 없습니다. (추상 메서드)
    void fly(); 
}

// 2️⃣ 비행기 클래스 (Flyable 계약에 서명함)
class Airplane implements Flyable {
    // 규격에 서명했으므로, 강제로 fly() 메서드의 알맹이를 짜야 합니다.
    @Override
    public void fly() {
        System.out.println("엔진이 점화됩니다!! 슈우우웅~ 비행기가 하늘을 납니다.");
    }
}

// 3️⃣ 독수리 클래스 (Flyable 계약에 서명함)
class Eagle implements Flyable {
    @Override
    public void fly() {
        System.out.println("시원하게 날개를 푸드덕 펼치고 독수리가 활공합니다.");
    }
}

// 본부석 (실행)
public class InterfaceMain {
    public static void main(String[] args) {
        
        // 다형성의 마법! Flyable(나는 능력을 가진 것) 바구니에 담기!
        Flyable f1 = new Airplane();
        Flyable f2 = new Eagle();
        
        // 비행기든, 독수리든, Flyable 약속을 지켰으니 무조건 fly() 버튼을 누를 수 있습니다!
        f1.fly(); 
        f2.fly(); 
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 메서드 구현 강제 회피: implements Flyable 이라고 선언해 놓고 만약 자신이 fly()의 속내용을 코딩하지 않으면, 자바는 "계약 위반!" 이라며 새빨간 에러를 계속 뿜어냅니다. 하나도 빠짐없이 오버라이딩 해야 합니다.
  2. 🚨 인터페이스 자체를 new 하기: Flyable f = new Flyable(); 은 절대 안 됩니다!!! 인터페이스는 실체가 없는 종이 쪼가리(약속)일 뿐입니다. 객체를 만들려면 반드시 그 종이에 서명한 실제 구현체(new Eagle() 등)를 지목해야 합니다.
  3. 🚨 접근 제어자 빼먹기: 인터페이스 안에 적힌 메서드는 자동으로 완전 공개(public) 취급을 받습니다. 그래서 구현 클래스에서 오버라이딩 할 때는 반드시 앞에 public void fly() 처럼 `public`을 명시해야 에러가 안 납니다!

[모듈 5: 실전 응용] 예외 처리(try-catch) - 에러 화면 대신 우아하게 방어하기

1. 💡 눈높이 비유 (The Analogy)

💡 배달 알바생과 예상치 못한 갑작스런 소나기!

배달 알바를 하다가 갑자기 폭우(에러/예외)가 쏟아졌습니다.
아무 대비가 없는 초보 배달원은 오토바이를 길거리에 팽개치고 오열하며 그 자리에서 배달(프로그램)을 완전히 중단(비정상 종료)합니다.

하지만 우비(try-catch)를 준비한 베테랑 배달원은 다릅니다!

  • 일단 배달을 시도해 봅니다. (try)
  • 만약 폭우가 쏟아지면 (예외 발생), 우비를 꺼내 입고 대처합니다. (catch)
  • 그리고 비가 오든 눈이 오든 배달을 '끝까지' 마칩니다. 프로그램이 중간에 공포의 빨간 글씨를 띄우고 뻗어버리는 걸 막는 든든한 보험입니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① try { ... } (시도하다)
    👉 "이 블록 안에는 '혹시나 에러가 날지도 모르는 아슬아슬한 코드(예: 숫자 나누기 0, 없는 파일 열기 등)'를 조심스럽게 감싸서 실행합니다."
  • ② catch (Exception e) { ... } (잡다, 대처하다)
    👉 "try 블록에서 폭탄(에러)이 터지면 그 즉시 실행이 멈추고 이 catch 블록으로 점프합니다. e.getMessage() 등으로 무슨 에러인지 파악하고, 사용자에게 '앗, 잘못된 입력입니다!' 라고 친절하게 안내합니다."
  • ③ finally { ... } (마무리하다)
    👉 "에러가 났든 안 났든, 무조건 100% 실행되어야 하는 정리 정돈 코드(예: 열어둔 파일 닫기)를 이곳에 넣습니다. (옵션이라 생략 가능)"

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

숫자를 0으로 나누는 치명적인 수학 에러(안드로메다행 계산!)를 우아하게 방어해 보겠습니다.

public class ExceptionMain {
    public static void main(String[] args) {
        
        System.out.println("프로그램 시작!");
        
        // 1️⃣ 에러가 발생할 가능성이 있는 코드를 try로 감싸줍니다.
        try {
            int result = 10 / 0; // 🚫 수학적으로 불가능! 여기서 폭탄(ArithmeticException) 터짐!
            
            // 폭탄이 터진 즉시 catch로 도망가기 때문에 아래 코드는 절대 실행되지 않습니다.
            System.out.println("계산 결과: " + result);
            
        } catch (ArithmeticException e) {
            // 2️⃣ 폭탄이 터졌을 때 실행되는 대피소(에러 처리) 부분
            System.out.println("🚨 삐빅! 숫자를 0으로 나눌 수 없어요.");
            // 개발자용 에러 이유 확인 (가장 친절한 에러 메시지)
            System.out.println("에러 사유: " + e.getMessage());
            
        } finally {
            // 3️⃣ 폭탄이 터지든 말든 무조건 거쳐가야 하는 마지막 출구
            System.out.println("☑️ 계산기 자원을 정리합니다.");
        }
        
        // 핵심! 프로그램이 도중에 죽지 않고 여기까지 무사히 도착해서 정상 종료됩니다.
        System.out.println("프로그램이 안전하게 종료되었습니다.");
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 빨간 줄(에러)과 예외(Exception)의 차이: 코드 작성 중 문법 틀려서 생기는 빨간 줄(컴파일 에러)은 try-catch로 못 잡습니다. 이건 실행 중에 갑자기 인터넷이 끊기거나 숫자에 문자를 검색하는 등의 '돌발 상황(런타임 에러)'을 잡는 용도입니다.
  2. 🚨 catch 창구 막기: 구체적인 에러(예: ArithmeticException)를 위에 쓰고, 젤 뭉뚱그려 잡는 만능 에러 Exception을 젤 밑(마지막 catch)에 써야 합니다. 만능 그물을 맨 위에 치면 아래 그물은 쓸모가 없어져 에러가 납니다.
  3. 🚨 에러 먹어버리기: catch 블록을 만들어 놓고 그 안에 아무것도 안 적어 두면, 어떤 에러가 났는지 전혀 알 수가 없어 나중에 디버깅할 때 미쳐버립니다! 최소한 System.out.println("에러발생"); 이라도 꼭 적어두세요.

import java.util.Scanner;

// --- [1. 부모 클래스: 모든 생명체의 기본 뼈대] ---
class Character {
    String name;
    int hp;
    int atk; // 공격력
    
    // 생성자
    public Character(String name, int hp, int atk) {
        this.name = name;
        this.hp = hp;
        this.atk = atk;
    }
    
    // 공격 메서드 (자식들이 입맛대로 바꿀 예정)
    public void attack(Character target) {
        System.out.println(name + "의 일반 공격!");
        target.takeDamage(atk);
    }
    
    // 피해를 입는 메서드
    public void takeDamage(int damage) {
        this.hp -= damage;
        System.out.println(name + "이(가) " + damage + "의 피해를 입었습니다! [HP: " + Math.max(0, hp) + "]");
    }
    
    public boolean isDead() {
        return hp <= 0;
    }
}

// --- [2. 자식 클래스: 영웅 (전사)] ---
class Hero extends Character {
    public Hero(String name) {
        super(name, 100, 20); // 부모 생성자 호출 (내 체력 100, 공격력 20)
    }
    
    // 오버라이딩: 영웅만의 멋진 공격
    @Override
    public void attack(Character target) {
        System.out.println("🗡️ [" + name + "]의 강려크한 검술 공격!!");
        target.takeDamage(atk);
    }
    
    // 특수 스킬 (회복)
    public void heal() {
        hp += 30;
        System.out.println("✨ [" + name + "]이(가) 응급처치로 체력을 30 회복했습니다! [HP: " + hp + "]");
    }
}

// --- [3. 자식 클래스: 몬스터] ---
class Monster extends Character {
    public Monster(String name, int hp, int atk) {
        super(name, hp, atk);
    }
    
    @Override
    public void attack(Character target) {
        System.out.println("👺 [" + name + "]의 매서운 발톱 할퀴기!!");
        target.takeDamage(atk);
    }
}

// --- [4. 본부석 (실행 및 게임 루프)] ---
public class RPGMain {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        System.out.println("=== 🔥 용사의 마을에 오신 것을 환영합니다! 🔥 ===");
        System.out.print("당신의 영웅 이름을 정해주세요: ");
        String heroName = sc.nextLine();
        
        // 인스턴스(객체) 생성
        Hero myHero = new Hero(heroName);
        Monster goblin = new Monster("흉폭한 고블린", 60, 10);
        
        System.out.println("\n야생의 [" + goblin.name + "]이(가) 나타났습니다! 전투 시작!");
        
        // 무한 반복 전투 흐름 (둘 중 하나가 죽을 때까지)
        while (true) {
            System.out.println("\n----------------------------------------");
            System.out.println("나의 HP: " + myHero.hp + " | 상대 HP: " + goblin.hp);
            System.out.println("1. 기본 공격   2. 방어 후 체력회복");
            System.out.print("행동을 선택하세요: ");
            
            int choice = Integer.parseInt(sc.nextLine());
            
            // 1턴: 플레이어의 행동
            if (choice == 1) {
                myHero.attack(goblin); // 내가 몬스터를 때림
            } else if (choice == 2) {
                myHero.heal(); // 회복
            } else {
                System.out.println("잘못된 선택입니다. 턴이 날아갔습니다!");
            }
            
            // 몬스터 사망 체크 (break로 루프 탈출)
            if (goblin.isDead()) {
                System.out.println("\n🏆 축하합니다! 고블린을 물리쳤습니다!");
                break;
            }
            
            // 2턴: 몬스터의 반격
            System.out.println("");
            goblin.attack(myHero);
            
            // 플레이어 사망 체크
            if (myHero.isDead()) {
                System.out.println("\n💀 눈앞이 깜깜해졌습니다... 게임 오버.");
                break;
            }
        }
        
        System.out.println("게임이 종료되었습니다. 수고하셨습니다!");
        sc.close();
    }
}
