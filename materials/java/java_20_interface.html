[모듈 4: 객체지향 심화] 인터페이스(Interface) - 약속과 규격 정하기 (리모컨의 비밀)

1. 💡 눈높이 비유 (The Analogy)

💡 컴퓨터의 USB 포트, 그리고 리모컨 규격!

우리가 삼성전자, LG전자, 다이소 등 어떤 브랜드의 마우스를 사더라도 그냥 'USB 단자 구멍'에 꽂으면 작동합니다. 왜 그럴까요?
전 세계 모든 마우스 회사들이 "USB라는 규격(약속)을 무조건 지키자!" 라고 서약서를 썼기 때문입니다.

인터페이스(Interface)가 바로 자바의 전 세계 공통 '규격'이자 철저한 '약속'입니다.

  • 추상적인 빈 껍데기 버튼(규격)만 만들어둡니다. (예: "모든 리모컨은 무조건 전원켜기 버튼을 가져야 해!")
  • 이 약속에 도장을 찍은(implements) 사람들은, 강제로 그 버튼을 어떻게 작동시킬지 각자 알아서 완벽하게 구현해야 합니다. (협업에 필수!)

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① interface 선언
    👉 "class 대신 interface 라고 적습니다. 이 안에는 몸통(중괄호 {}) 없는 텅 빈 메서드(추상 메서드)들만 가득 나열해서 규격만 적립합니다."
  • ② implements (구현하다)
    👉 상속이 extends 였다면, 인터페이스는 implements를 씁니다. "나 클래스 A는, 이 인터페이스 규격을 완벽하게 내 몸속에 구현해서 작동시키겠다!" 라는 강제 서약서입니다.
  • ③ 다중 구현 허용!
    👉 "부모(상속)는 오직 1명만 모실 수 있지만, 인터페이스는 규격일 뿐! 그래서 '나는 USB 규격도 따르고(implements), 블루투스 규격도 따를래!' 라며 implements A, B, C 처럼 무한대로 약속에 가입할 수 있습니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

'비행 기능(Flyable)' 이라는 자격증(약속) 규격을 만들고, 비행기와 새가 이를 어떻게 구현하는지 봅시다.

// 1️⃣ 규격 설계도 (인터페이스)
// "이 규격에 가입하고 싶으면, 반드시 날아오르는 fly() 기술을 만들어야 해!"
interface Flyable {
    // 껍데기만 존재함! { 블록 } 이 절대 없습니다. (추상 메서드)
    void fly(); 
}

// 2️⃣ 비행기 클래스 (Flyable 계약에 서명함)
class Airplane implements Flyable {
    // 규격에 서명했으므로, 강제로 fly() 메서드의 알맹이를 짜야 합니다.
    @Override
    public void fly() {
        System.out.println("엔진이 점화됩니다!! 슈우우웅~ 비행기가 하늘을 납니다.");
    }
}

// 3️⃣ 독수리 클래스 (Flyable 계약에 서명함)
class Eagle implements Flyable {
    @Override
    public void fly() {
        System.out.println("시원하게 날개를 푸드덕 펼치고 독수리가 활공합니다.");
    }
}

// 본부석 (실행)
public class InterfaceMain {
    public static void main(String[] args) {
        
        // 다형성의 마법! Flyable(나는 능력을 가진 것) 바구니에 담기!
        Flyable f1 = new Airplane();
        Flyable f2 = new Eagle();
        
        // 비행기든, 독수리든, Flyable 약속을 지켰으니 무조건 fly() 버튼을 누를 수 있습니다!
        f1.fly(); 
        f2.fly(); 
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 메서드 구현 강제 회피: implements Flyable 이라고 선언해 놓고 만약 자신이 fly()의 속내용을 코딩하지 않으면, 자바는 "계약 위반!" 이라며 새빨간 에러를 계속 뿜어냅니다. 하나도 빠짐없이 오버라이딩 해야 합니다.
  2. 🚨 인터페이스 자체를 new 하기: Flyable f = new Flyable(); 은 절대 안 됩니다!!! 인터페이스는 실체가 없는 종이 쪼가리(약속)일 뿐입니다. 객체를 만들려면 반드시 그 종이에 서명한 실제 구현체(new Eagle() 등)를 지목해야 합니다.
  3. 🚨 접근 제어자 빼먹기: 인터페이스 안에 적힌 메서드는 자동으로 완전 공개(public) 취급을 받습니다. 그래서 구현 클래스에서 오버라이딩 할 때는 반드시 앞에 public void fly() 처럼 `public`을 명시해야 에러가 안 납니다!

4. 📝 스스로 챌린지 (Mini Challenge)

소리를 재생할 수 있는 장치들이 지켜야 할 Playable 인터페이스를 만들고 적용해 보세요!

📝 미션: 다양한 음악 기기 제어하기

1. Playable 이라는 인터페이스를 만들고, 텅 빈 void play(); 메서드를 선언하세요.
2. MP3Player 클래스를 만들고 implements Playable로 연결한 뒤, "MP3에서 신나는 음악이 나옵니다 둠칫둠칫!" 을 출력하도록 오버라이딩하세요.
3. 본부석에서 Playable device = new MP3Player(); 형태로 객체를 만들고 device.play();를 실행해 보세요!

👉 힌트 & 정답(클릭해서 확인하세요)
정답 코드: 
// 재생 규격서
interface Playable {
    void play(); // 빈 껍데기 버튼 설계도
}

// MP3 플레이어 실체 생성
class MP3Player implements Playable {
    @Override
    public void play() {
        System.out.println("💽 MP3에서 신나는 음악이 재생됩니다. 둠칫둠칫 파뤼타임~");
    }
}

public class Main {
    public static void main(String[] args) {
        // 규격(리모컨)으로 기기 전원을 켬
        Playable device1 = new MP3Player();
        device1.play();
    }
}
