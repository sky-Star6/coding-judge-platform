[모듈 1: 기초 탄탄] 조건문 1단계 (if, else if, else)

1. 💡 눈높이 비유 (The Analogy)

💡 내비게이션의 갈림길 선택하기!

지금까지의 컴퓨터는 위에서 아래로 무조건 한 뱡항으로만 달리는 직진 롤러코스터였습니다.

하지만 조건문(Condition)을 배우는 순간, 컴퓨터는 선택의 갈림길을 지닌 '내비게이션'으로 진화합니다!

  • **if (만약에 ~라면):** "만약 비가 온다면(조건), 우산을 챙긴다(행동)!"
  • **else if (그게 아니고, 또 다른 만약에):** "비는 안 오는데 눈이 온다면(조건2), 패딩을 챙긴다(행동2)!"
  • **else (위의 모든 경우가 아니라면):** "비도 안 오고 눈도 안 오면, 그냥 나간다!(기본 행동)"

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① if 구문
    👉 "괄호 ( ) 안의 내용이 참(true)일 때만 중괄호 { } 안방으로 들어가서 코드를 실행하는 철통 보안 경비원입니다."
  • ② else 구문
    👉 "if의 검문을 통과하지 못한(false) 모든 떨거지들을 다 받아주는 최후의 보루(마지막 도착지)입니다. 괄호로 조건식을 적지 않습니다!"
  • ③ 코드 블록 { }
    👉 "이 조건이 맞았을 때, 과연 어디서부터 어디까지 실행할 것인가? 그 범위를 지정해주는 울타리입니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

나이에 따라 버스 요금 등급을 나눠주는 프로그램을 만들어 볼까요?

public class ConditionIf {
    public static void main(String[] args) {
        
        int age = 15; // 내 나이
        
        // 1️⃣ 만약 나이가 19살 이상(성인) 이라면?
        if (age >= 19) {
            System.out.println("성인입니다. 삐빅- 1200원");
        }
        // 2️⃣ 성인이 아니고, 그게 아니라 만약 나이가 13살 이상(청소년) 이라면?
        else if (age >= 13) {
            System.out.println("청소년입니다. 삐빅- 720원"); // age가 15니까 이곳이 실행됩니다!
        }
        // 3️⃣ 위의 모든 조건이 아니라면? (어린이)
        else {
            System.out.println("어린이입니다. 삐빅- 450원");
        }
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 if 뒤에 세미콜론(;) 테러: if (age > 20); { .. } 괄호 뒤에 마침표(;)를 쾅 찍어버리면, 조건문이 그 자리에서 허무하게 종결돼버립니다! 중괄호가 무조건 실행되는 무서운 버그가 숨어버리니 조심하세요.
  2. 🚨 else 뒤에 억지로 조건 달기: else (age < 13) { .. }. else는 남은 모든 방어막이므로, 절대 괄호( ) 조건을 달 수 없습니다. 조건이 필요하면 무조건 else if를 쓰세요!
  3. 🚨 순서의 반란: 위에서 아래로 물이 흐르듯 검사합니다! 만약 if(age>=13) 청소년 을 if(age>=19) 성인 보다 먼저 배치하면? 20살도 13이상이니 청소년으로 걸려서 통과해버리는 대형 사고가 납니다! (항상 좁고 구체적인 조건부터 먼저 쓰세요)
