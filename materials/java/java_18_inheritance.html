[모듈 4: 객체지향 심화] 상속(Inheritance) - 부모의 유산을 그대로 물려받기

1. 💡 눈높이 비유 (The Analogy)

💡 백종원의 만능 간장 베이스!

요리를 할 때 떡볶이, 제육볶음, 찜닭을 처음부터 끝까지 다 따로 만들면 양념(코드)을 3번 만들어야 해서 너무 귀찮습니다.

객체지향 프로그래밍의 핵사이다, 상속(Inheritance)은 마치 '백종원표 만능 간장(부모 클래스)'을 만들어 두는 것과 같습니다.

  • 만능 간장 베이스(부모)를 만들어 두고, 여기서 확장(상속)하여 파만 넣으면 '찜닭 소스(자식 1)'가 되고, 고추장을 추가하면 '제육 소스(자식 2)'가 됩니다.
  • 기본 메뉴얼(코드)을 복사/붙여넣기 할 필요 없이, 부모의 모든 재산(필드)과 기술(메서드)을 그냥 꿀꺽! 편하게 얻어오고 내 것만 덧붙이면 끝입니다. 중복 코드가 확 줄어들죠!

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 상속 키워드 extends (확장하다)
    👉 "자식 클래스 이름 뒤에 extends 부모클래스 라고 적습니다. 자바에서는 상속을 '받는다'기보단, 부모를 발판 삼아 내 능력을 더 '확장(extends)'한다는 아주 긍정적인 의미로 씁니다."
  • ② 부모/슈퍼 클래스 (Super Class)
    👉 "유산을 물려주는 쪽입니다. (예: 스마트폰, 동물, 자동차 같은 큼직큼직하고 공통적인 뼈대)"
  • ③ 자식/서브 클래스 (Sub Class)
    👉 "유산을 받는 쪽입니다. 부모의 능력을 기본 패시브로 다 쓰면서 자기만의 특수 능력도 가집니다. (예: 아이폰, 고양이, 스포츠카 같은 세부적인 실체)"

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

기본 스마트폰(부모)의 능력을 물려받고 애플페이 특수능력을 추가한 아이폰(자식)을 만들어 봅시다.

// 1️⃣ 만능 베이스! 부모 클래스: 스마트폰
class Phone {
    String model;    // 기종
    String number;   // 전화번호
    
    // 부모의 공통 기술 (물려줄 유산)
    void call() {
        System.out.println(number + " 번호로 전화를 겁니다. 따르릉~");
    }
}

// 2️⃣ 자식 클래스: 아이폰 (Phone의 모든 능력을 '확장'해서 가져옴!)
class IPhone extends Phone {
    
    // ⚠️ 주목! 자식 클래스 안에는 call(), model 같은 걸 또 적을 필요가 전혀 없습니다! 이미 내 몸 안에 다 있습니다.
    
    // 오직 나만의 특수 능력(알파)만 추가합니다.
    void applePay() {
        System.out.println(model + " 모델로 애플페이 띠링! 결제가 완료되었습니다.");
    }
}

// 본부석 (실행)
public class ExtendMain {
    public static void main(String[] args) {
        
        // 아이폰 하나 샀습니다.
        IPhone myPhone = new IPhone();
        
        // 1. 부모가 물려준 변수 사용 가능!
        myPhone.model = "아이폰 15";
        myPhone.number = "010-1111-2222";
        
        // 2. 부모가 물려준 통화 기술 사용 가능!
        myPhone.call(); 
        
        // 3. 내 고유 기술인 애플페이 사용 가능!
        myPhone.applePay(); 
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 다중 상속 불가능: 엄마의 능력도 받고 싶고 아빠의 능력도 받고 싶다며 extends Mom, Dad 처럼 두 부모를 동시에 적으면 자바는 즉시 에러를 뿜어냅니다! 자바에서 부모 클래스는 무조건 단 1개만 지목할 수 있습니다.
  2. 🚨 private 유산은 물려줄 수 없음: 부모 거북이가 private int secretGold = 1000; 이라고 꽁꽁 숨겨둔 무덤 속 비자금은 자식이라도 절대 접근 불가입니다. 물려주고 싶다면 public이나 protected를 써야 합니다.
  3. 🚨 의미 없는 남용 주의: 그랜저에다가 억지로 냉장고를 `extends` 시키면 안 됩니다! 상속은 반드시 "A는 B이다(A is a B)" 관계가 논리적으로 성립할 때만 써야 합니다. (예: "아이폰은 스마트폰이다." ➔ OK!)

4. 📝 스스로 챌린지 (Mini Challenge)

기본 TV를 상속받아 스마트TV(SmartTV)를 디자인해 보세요!

📝 미션: 스마트 TV 업그레이드

1. 부모 클래스 TV를 만드세요.
- 필드: boolean power (전원상태)
- 기능: void turnOn() (전원을 true로 켜고 "TV 전원이 켜졌습니다." 출력)

2. 자식 클래스 SmartTV를 만들고 extends TV로 상속받으세요.
- 특수 기능: void netflix() ("넷플릭스 애플리케이션을 실행합니다." 출력)

3. 본부석(main)에서 SmartTV 객체를 한 대 장만하고, 부모 기술로 전원을 먼저 켠 뒤, 자식 기술로 넷플릭스를 실행해 보세요!

👉 힌트 & 정답(클릭해서 확인하세요)
정답 코드: 
// 부모 TV
class TV {
    boolean power;
    
    void turnOn() {
        power = true;
        System.out.println("TV 전원이 켜졌습니다. 환영합니다.");
    }
}

// 자식 스마트 TV
class SmartTV extends TV {
    // 속성, turnOn() 등은 안 적었지만 이미 물려받은 상태!
    
    void netflix() {
        if (power == true) {
            System.out.println("넷플릭스 애플리케이션을 실행합니다. 둥~둥~");
        } else {
            System.out.println("먼저 부모 리모컨으로 전원을 켜주세요!");
        }
    }
}

public class Main {
    public static void main(String[] args) {
        SmartTV myRoomTv = new SmartTV();
        
        myRoomTv.turnOn();  // 부모님 기능!
        myRoomTv.netflix(); // 내 고유 기능!
    }
}
