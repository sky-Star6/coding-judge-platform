[모듈 3: 클래스와 객체] 클래스(Class)와 객체(Object)의 개념 - 붕어빵 틀과 붕어빵

1. 💡 눈높이 비유 (The Analogy)

💡 붕어빵을 끝없이 구워내는 마법의 틀!

지금까지는 int, String 같은 기본 재료만으로 프로그램을 짰습니다. 하지만 복잡한 시스템(예: 은행, 게임)을 만들려면 나만의 복합적인 재료 세트가 필요합니다.

이것이 바로 객체지향 프로그래밍(OOP)의 시작, 클래스와 객체입니다!

  • **클래스(Class): 붕어빵 틀 (설계도)**
    "머리부터 꼬리까지 이런 모양이고, 안에는 팥이 들어간다!"라고 그려놓은 도면이나 거푸집입니다. 눈으로 볼 수 있는 실체는 아니고, 단지 '설명서'일 뿐입니다.
  • **객체(Object / Instance): 만들어진 붕어빵 (실체)**
    설계도를 바탕으로 밀가루를 부어 구워낸 실제 먹을 수 있는 붕어빵입니다. 팥 붕어빵 한 개, 슈크림 붕어빵 한 개, 치즈 붕어빵 한 개... 이 도면(Class) 하나만 있으면 실체(Object)를 100개든 1000개든 무한으로 찍어낼 수 있습니다!

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 클래스 선언 ( class 이름 {} )
    👉 "class Dog 라고 적으면, '이제부터 컴퓨터 안에 나만의 반려견을 만드는 틀을 정의하겠다!' 라는 선언입니다."
  • ② 인스턴스화 ( new 클래스명() )
    👉 "붕어빵 틀에서 구워내는 행동을 '인스턴스화'라고 합니다. new Dog() 라고 마법의 주문을 외치면, 메모리라는 쟁반 위에 강아지 형체 1마리가 뿅! 하고 진짜로 생겨납니다."
  • ③ 객체 참조 변수 ( 클래스명 변수명 = new ... )
    👉 "생성된 강아지는 이리저리 도망갈 수 있습니다. 그래서 Dog myMax = new Dog() 처럼, myMax라는 이름의 목줄(참조 변수)에 묶어두어야 우리가 마음대로 부를 수 있습니다."

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

우리만의 자동차 공장을 짓고(Class), 실제 자동차 두 대를 조립(Object)해 봅시다!

// 1️⃣ 자동차(Car) 설계도(클래스)를 만듭니다. (실제 차가 아님!)
class Car {
    // 자동차가 가지는 상태 (속성)
    String modelName; // 모델명
    String color;     // 색상
    
    // 자동차가 할 수 있는 행동 (메서드)
    void honk() {
        System.out.println(modelName + " : 빵빵! 비키세요!");
    }
}

// 프로그램을 실행하는 본부석 (Main 클래스)
public class FactoryMain {
    public static void main(String[] args) {
        
        // 2️⃣ 첫 번째 자동차 조립(생성)! - new 키워드 사용
        Car car1 = new Car(); 
        car1.modelName = "소나타"; // 점(.)을 찍어서 속성에 접근
        car1.color = "검정색";
        
        // 3️⃣ 두 번째 자동차 조립(생성)! - 틀은 하나지만 여러 개 무한 생산 가능
        Car car2 = new Car(); 
        car2.modelName = "모닝";
        car2.color = "노란색";
        
        // 4️⃣ 각 자동차에게 행동 명령 내리기
        car1.honk(); // 소나타 : 빵빵! 비키세요!
        car2.honk(); // 모닝 : 빵빵! 비키세요!
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 설계도랑 실체를 헷갈림: Car.color = "빨강"; 형태로 틀(Class) 자체에 색칠을 하려고 하면 에러가 납니다! 색칠은 반드시 new Car()로 구워낸 실체(객체)인 car1.color = "빨강"; 에다가 해야 합니다.
  2. 🚨 new 없이 이름만 부르기: Car car3; 라고만 적으면 차가 생긴 게 아닙니다! 주차 공간(변수)만 잡아두었을 뿐, 안에 텅 빈 null 상태입니다. 반드시 new Car(); 까지 써줘야 실체가 만들어집니다!
  3. 🚨 대소문자 관례 어기기: 클래스의 첫 글자는 반드시 대문자(Car, Dog)로 쓰는 것이 전 세계 개발자들의 공통 약속입니다. 소문자(car, dog)로 쓰면 일반 변수와 구별이 안 되어 매우 혼란스럽습니다.

3. 본부석(main 메서드) 안에서 new 키워드를 사용해 여러분의 고양이나 강아지 객체를 1마리 생성하세요.
4. 생성된 객체의 이름과 나이에 값을 넣고, 화면에 "나의 펫 이름은 야옹이 이고, 3살입니다." 라고 출력해 보세요.

👉 힌트 & 정답(클릭해서 확인하세요)
정답 코드: 
// 1. 클래스(틀) 만들기
class Animal {
    String name;
    int age;
}

public class Main {
    public static void main(String[] args) {
        // 2. 객체(실체) 만들기
        Animal myPet = new Animal();
        
        // 3. 속성에 값 채워넣기
        myPet.name = "야옹이";
        myPet.age = 3;
        
        // 4. 출력하기
        System.out.println("나의 펫 이름은 " + myPet.name + " 이고, " + myPet.age + "살입니다.");
    }
}
