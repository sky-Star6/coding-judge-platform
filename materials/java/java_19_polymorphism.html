[모듈 4: 객체지향 심화] 다형성(Polymorphism)과 오버라이딩 - 내 입맛대로 부모 능력 바꾸기!

1. 💡 눈높이 비유 (The Analogy)

💡 카멜레온의 변신술과 "아빠, 난 좀 다르게 할래!"

다형성(Polymorphism)이란 '하나의 객체가 여러 가지 형태를 가질 수 있는 성질'을 뜻합니다. 어려어 보이만 실생활에서 흔히 쓰입니다.

  • **다형성:** 동물병원 의사가 "다음 동물(Animal) 들어오세요~ 진찰할게요!" 라고 했을 때, 강아지가 들어와도 진찰() 가능하고, 고양이가 들어와도 진찰()이 가능합니다. 모두 '동물'이라는 부모 이름표로 묶어 부를 수 있기 때문입니다.
  • **오버라이딩(Overriding):** 부모(Animal)가 물려준 울음소리내기() 기술을 물려받았는데, 강아지(자식)는 "난 멍멍이라고 할래!", 고양이(자식)는 "난 야옹이라고 덮어쓸래!" 라며 부모의 기술(메서드)을 자기 입맛대로 뜯어 고치는(덮어쓰기) 것입니다.

2. 🔑 핵심 키워드 정리 (Key Terms)

🔑 필수 키워드 3가지
  • ① 오버라이딩 (Overriding = 덮어쓰기)
    👉 "부모 클래스에 이미 있는 메서드를, 자식 클래스에서 똑같은 이름, 똑같은 괄호() 형태로 다시 적어서 내용물만 교체하는 겁니다."
  • ② @Override 어노테이션
    👉 "메서드 바로 위에 달아두는 이름표입니다. 이거 부모 메서드 덮어쓴 거예요! 라고 컴퓨터(컴파일러)에게 알려줘서, 혹시 내가 오타를 냈다면 바로 에러를 띄워 짚어주는 고마운 보호막입니다."
  • ③ 다형적 형변환 ( 부모타입 변수 = new 자식() )
    👉 "Animal a = new Dog(); 라고 쓸 수 있는 마법입니다. 개(Dog)는 일종의 동물(Animal)이므로, 넓은 의미의 동물 바구니에 개를 담아둘 수 있습니다!"

3. 💻 실습 코드 & 상세 분석 (Code Snippet)

모든 동물(부모)은 소리를 낼 수 있습니다. 하지만 이 유산을 각 동물(자식)들끼리 어떻게 다르게 사용하는지 볼까요?

// 1️⃣ 부모 클래스: 동물 베이스
class Animal {
    // 보편적인 동물의 울음소리 (유산)
    void makeSound() {
        System.out.println("...동물이 소리를 냅니다...");
    }
}

// 2️⃣ 자식 클래스: 개 (부모의 소리를 멍멍!으로 조작)
class Dog extends Animal {
    // 부모의 makeSound()를 내 방식대로 덮어씁니다!
    @Override 
    void makeSound() {
        System.out.println("왈왈!! 멍멍!!");
    }
}

// 3️⃣ 자식 클래스: 고양이 (부모의 소리를 야옹!으로 조작)
class Cat extends Animal {
    @Override 
    void makeSound() {
        System.out.println("야~~~옹 냐아아앙");
    }
}

// 본부석
public class PolyMain {
    public static void main(String[] args) {
        
        // --- [다형성의 마법] ---
        // 부모 이름표(Animal) 하나로 여러 자식들을 일렬로 세울 수 있습니다!
        Animal myDog = new Dog(); 
        Animal myCat = new Cat();
        
        // 똑같이 "소리내봐!" (makeSound) 라고 불렀는데, 대답이 각기 다릅니다!
        myDog.makeSound(); // 출력: 왈왈!! 멍멍!!
        myCat.makeSound(); // 출력: 야~~~옹 냐아아앙
        
    }
}
[주의!] ⚠️ 입문자 단골 실수 탑(TOP) 3
  1. 🚨 오버라이딩 vs 오버로딩 혼동:
    - 오버라이딩(Overriding): 부모 클래스에 올라타서 부모의 것을 덮어쓰기 (서로 다른 클래스 간)
    - 오버로딩(Overloading): 내 클래스 안에서 여러 개체를 담기 위해 매개변수만 바꿔서 다양하게 쌓아 올리기 (같은 클래스 안) 이 두 개념은 완전히 다릅니다!
  2. 🚨 메서드 이름 오타: 부모는 makeSound()인데 자식이 실수로 makesound()(소문자 s)로 만들면, 자바는 덮어쓰기가 아니라 '아, 새로운 메서드 만들었구나' 하고 전혀 다른 취급을 합니다. @Override 어노테이션을 쓰면 이런 오타를 막아줍니다!
  3. 🚨 반대로 담기: Animal a = new Dog()는 가능하지만, Dog d = new Animal()은 불가능합니다! "모든 강아지는 동물이지만, 모든 동물이 다 강아지는 아니다" 라는 논리를 꼭 기억하세요.

4. 📝 스스로 챌린지 (Mini Challenge)

다양한 종류의 피자(Pizza) 클래스들이 부모의 능력을 어떻게 다르게 쓰는지 구현해 봅시다!

📝 미션: 피자 맛집, 종류마다 다른 토핑 자랑!

1. 부모 Pizza 클래스를 만들고, void bake() { System.out.println("피자를 굽습니다."); } 메서드를 만드세요.
2. 그것을 상속받는 자식 CheesePizza 클래스를 만드세요. 그리고 bake() 메서드를 오버라이딩(덮어쓰기)하여 "치즈가 쭈욱 늘어나는 피자를 굽습니다." 로 변경하세요.
3. 본부석에서 Pizza myPizza = new CheesePizza(); 라고 다형성을 이용해 객체를 만든 뒤, myPizza.bake(); 를 호출해 결과를 확인하세요.

👉 힌트 & 정답(클릭해서 확인하세요)
정답 코드: 
// 부모 피자
class Pizza {
    void bake() {
        System.out.println("일반 밀가루 도우 피자를 굽습니다.");
    }
}

// 자식 치즈 피자
class CheesePizza extends Pizza {
    @Override
    void bake() {
        System.out.println("치즈가 쭈욱~ 완벽하게 늘어나는 치즈크러스트 피자를 굽습니다!");
    }
}

public class Main {
    public static void main(String[] args) {
        // 피자 껍데기(참조타입)로 감싸지만, 실제 알맹이는 치즈피자!
        Pizza p = new CheesePizza();
        
        p.bake(); // 출력 결과: 자식이 덮어쓴 "치즈가 쭈욱~..." 이 나옴!
    }
}
