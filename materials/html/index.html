🌐 1. 웹의 세계로 출발!

웹사이트는 대체 어떻게 만들어져 있을까요?

1️⃣ 웹을 구성하는 3대장

우리가 매일 보는 네이버, 유튜브, 인스타그램 같은 웹사이트들은 모두 세 가지 언어의 합작품입니다: HTML, CSS, JavaScript.

🏠 건물을 짓는 과정에 비유해 볼까요?
  • HTML (구조): 건물의 뼈대와 철근. 방은 어디에 있고, 문은 어디인지 구조를 잡습니다.
  • CSS (디자인): 건물의 인테리어. 벽지 색깔은 무엇인지, 조명은 어떻게 달지 꾸밉니다.
  • JavaScript (동작): 건물의 전기/수도 시설. 버튼을 누르면 엘리베이터가 움직이고, 물이 나오게 합니다.

📊 한눈에 보기

언어 역할 예시 (사람에 비유)
HTML (HyperText Markup Language) 뼈대 잡기, 정보의 내용 💀 사람의 뼈와 뼈대
CSS (Cascading Style Sheets) 꾸미기, 예쁘게 배치하기 👕 옷차림, 화장, 헤어스타일
JavaScript (JS) 동작, 상호작용(인터랙션) 🏃 근육, 신경계 (움직임)

2️⃣ 웹 브라우저의 마법

HTML 파일은 사실 영어로 쓰여진 단순한 텍스트 파일(메모장)입니다. 이 텍스트를 화려한 웹페이지로 그림을 그려주는 화가가 바로 크롬(Chrome), 사파리, 엣지 같은 웹 브라우저입니다.

<!-- 개발자가 쓴 암호 같은 텍스트 -->
<h1>환영합니다!</h1>
<button>클릭하세요</button>

위 코드를 브라우저가 읽으면, 화면에 크고 진한 글씨체로 "환영합니다!"를 쓰고, 마우스로 누를 수 있는 진짜 "버튼"을 만들어냅니다.

🎯 웹 기초 확인 미션

다음 중 웹사이트에서 버튼을 눌렀을 때 팝업창이 뜨는 '동작'을 담당하는 기술은 무엇일까요?

1. HTML (HyperText Markup Language)
2. CSS (Cascading Style Sheets)
3. JavaScript
4. Chrome 브라우저

📌 핵심 요약

  • HTML: 웹페이지의 내용과 뼈대를 만듭니다. (내용)
  • CSS: 웹페이지를 예쁘게 꾸밉니다. (디자인)
  • JavaScript: 웹페이지가 살아서 움직이게 합니다. (동작)
  • 웹 브라우저: 코드를 읽어 우리가 보는 화면으로 그려주는 도구입니다.

📄 2. 첫 HTML 페이지 만들기

태그(Tag)의 비밀과 HTML 문서의 기본 구조

1️⃣ HTML의 핵심: 태그 (Tag)

HTML은 내용물 앞뒤에 꼬리표(Tag)를 붙여서, 브라우저에게 "이건 제목이야", "이건 버튼이야" 하고 알려주는 언어입니다.

🏷️ 태그는 샌드위치 빵!
태그는 항상 여는 태그(빵 위쪽)와 닫는 태그(빵 아래쪽)가 세트로 다닙니다. 그리고 그 사이에 내용물(햄, 치즈)을 넣어요!
<태그이름> 내용물 </태그이름> (닫는 태그에는 꼭 / 가 들어갑니다) 
<h1>나는 가장 큰 제목입니다</h1>
<p>나는 평범한 문단(Paragraph)입니다.</p>
<button>클릭</button>

2️⃣ HTML 문서의 기본 뼈대

모든 HTML 문서는 꼭 지켜야 하는 규칙적인 구조가 있습니다.

👤 HTML 문서는 사람의 몸!
- 머리(Head): 눈에 보이지 않는 정보 (생각, 이름표)
- 몸통(Body): 눈에 보이는 실제 모습 (옷, 피부) 
<!DOCTYPE html> <!-- 나는 HTML5 최신 버전으로 쓰였어! 라고 선언 -->
<html lang="ko"> <!-- 전체 HTML 문서의 시작 (언어는 한국어) -->

  <head> <!-- 브라우저에게 주는 정보 (화면엔 안 보임) -->
    <meta charset="UTF-8"> <!-- 한글이 깨지지 않게 해줘! -->
    <title>나의 첫 웹페이지</title> <!-- 브라우저 탭에 뜨는 제목 -->
  </head>

  <body> <!-- 사용자가 화면에서 직접 보는 내용들 -->
    <h1>안녕, 웹 브라우저!</h1>
    <p>드디어 나만의 첫 웹페이지를 만들었어요.</p>
  </body>

</html> <!-- HTML 문서의 끝 -->

👨‍💻 직접 쳐보고 눈으로 확인하기

아래 박스에 코드를 입력하면 즉시 결과가 오른쪽에 나타납니다. <body> 안쪽 내용을 자유롭게 바꿔보세요!

⚡ HTML 실시간 테스트기

🎯 구조 확인 미션

우리가 눈으로 보는 실제 콘텐츠(글, 이미지, 버튼 등)는 어떤 태그 안에 작성해야 할까요?

1. <head> 태그 안
2. <body> 태그 안
3. <title> 태그 안
4. <html> 태그 밖

📌 핵심 요약

  • 태그: 여는 태그 <tag> 와 닫는 태그 </tag> 가 한 세트입니다.
  • <!DOCTYPE html>: 이 문서는 HTML5로 작성되었다는 선언입니다.
  • <head>: 브라우저를 위한 숨겨진 설정 정보가 들어갑니다. (탭 제목, 인코딩 등)
  • <body>: 브라우저 화면에 직접 그려지는 모든 내용이 들어갑니다.

📝 3. 텍스트 태그 마스터

글자를 크고, 굵게, 기울이게 만들어 봐요!

1️⃣ 제목 (Heading) 태그: h1 ~ h6

뉴스 기사를 보면 가장 큰 헤드라인이 있고, 그 아래 작은 소제목들이 있죠? HTML에서도 h1부터 h6까지 제목의 크기와 중요도를 6단계로 나눌 수 있습니다.

⚡ H 태그 테스트

⚠️ 주의: <h1>은 한 페이지에 한 번만 사용하는 것이 검색 엔진(구글, 네이버)에게 좋습니다!

2️⃣ 문단 (Paragraph)과 줄바꿈 태그

긴 글을 쓸 때는 <p> 태그를 사용합니다. 주의할 점은 HTML 코드에서 엔터(Enter)를 백 번 쳐도 브라우저는 한 칸 띄어쓰기로만 인식합니다.

✂️ 강제 줄바꿈 <br>
줄을 바꾸고 싶을 때는 엔터 키 대신 <br> 태그를 써야 합니다.
특이하게도 <br>은 닫는 태그(</br>)가 없는 빈 태그(Empty Tag)입니다! (빵 껍질 없이 소시지만 있는 핫도그랄까요?) 
<p>
이 문장은 코드에서 
엔터를 쳤지만 
화면에서는 한 줄로 나옵니다.
</p>

<p>
이 문장은 <br>
br 태그를 사용해서 <br>
세 줄로 나옵니다!
</p>

3️⃣ 글자 강조하기

문장 안에서 특정 단어를 굵게 하거나 기울이고 싶을 때 사용합니다.

  • <strong>: 매우 중요한 텍스트를 굵게 표시합니다.
  • <em>: 텍스트를 기울여서(Italic) 강조합니다.
  • <hr>: 가로줄을 쫙 그어줍니다. (마찬가지로 닫는 태그가 없습니다!)
⚡ 강조 태그 테스트

📌 핵심 요약

  • <h1> ~ <h6>: 제목을 나타냅니다. 1이 가장 큽니다.
  • <p>: 하나의 문단(단락)을 만듭니다.
  • <br>: 줄바꿈을 합니다. (닫는 태그 없음!)
  • <hr>: 수평 가로줄을 긋습니다. (닫는 태그 없음!)
  • <strong>, <em>: 글자를 굵게 하거나 기울여서 강조합니다.

📋 4. 목록 (List) 만들기

점박이 목록과 숫자 목록, 완벽하게 정리하기!

1️⃣ 순서 없는 목록 (Unordered List)

순서가 중요하지 않은 항목들(예: 쇼핑 리스트, 준비물)을 나열할 때 사용합니다. 항목 앞에 예쁜 점(Bullet)이 붙습니다.

  • <ul>: "지금부터 순서 없는 목록을 시작할게!" (Unordered List)
  • <li>: "이게 첫 번째 항목이야" (List Item)
<h3>마트 쇼핑 리스트</h3>
<ul>
  <li>사과</li>
  <li>우유</li>
  <li>초코파이</li>
</ul>

2️⃣ 순서 있는 목록 (Ordered List)

레시피나 순위처럼 순서가 아주 중요한 항목들을 나열할 때 사용합니다. 항목 앞에 1, 2, 3 숫자가 자동으로 붙습니다.

  • <ol>: "지금부터 번호가 달린 목록을 시작할게!" (Ordered List)
  • <li>: "항목은 똑같이 li를 써!" (List Item)
<h3>라면 끓이는 순서</h3>
<ol>
  <li>물을 끓인다.</li>
  <li>스프와 면을 넣는다.</li>
  <li>3분 기다린다.</li>
</ol>

👨‍💻 직접 리스트 만들어보기

⚡ 목록 태그 테스트

🎯 목록 태그 미션

순서가 있는 숫자 목록(1, 2, 3...)을 만들 때 가장 바깥을 감싸야 하는 태그는 무엇일까요?

1. <ul>
2. <li>
3. <ol>
4. <list>

📌 핵심 요약

  • <ul>: 점(Bullet)이 찍히는 순서 없는 목록 (Unordered)
  • <ol>: 숫자(1, 2, 3)가 달리는 순서 있는 목록 (Ordered)
  • <li>: ul이나 ol 안에 들어가는 개별 항목 (List Item)
  • *반드시 ul이나 ol 태그 안에는 li 태그만 직접 들어가야 합니다!

🔗 5. 링크와 이미지

웹(Web)을 거미줄처럼 이어주는 마법의 태그!

1️⃣ 다른 페이지로 이동! 링크 (Link) 태그

글자나 이미지를 클릭했을 때 다른 페이지로 넘어가게 해주는 <a>(Anchor) 태그입니다. (하이퍼텍스트의 꽃이죠!)

🧭 속성 (Attribute)의 등장!
단순히 "이건 링크야" 라고 말하면 컴퓨터는 "어디로 가라고?"라고 되물을 겁니다.
그래서 태그 안쪽에 href="주소" 처럼 추가적인 정보(속성)를 적어줘야 합니다. 
<!-- href 속성에 이동할 인터넷 주소를 적습니다 -->
<a href="https://www.google.com">구글로 이동하기</a>

🚪 새 창에서 열기 (target 속성)

현재 페이지는 놔두고, 새 탭에서 링크를 열고 싶다면 target="_blank" 속성을 추가하세요.

<a href="https://www.naver.com" target="_blank">새 탭에서 네이버 열기</a>

2️⃣ 화면에 사진 띄우기! 이미지 (Image) 태그

웹페이지에 예쁜 사진을 넣고 싶을 때는 <img> 태그를 사용합니다. 이 태그도 내용물이 필요 없어서 닫는 태그가 없는 빈 태그입니다!

  • src 속성 (Source): 이미지 파일이 있는 주소(경로)를 적습니다.
  • alt 속성 (Alternative text): 이미지가 깨졌을 때 대신 보여줄 글자, 또는 시각장애인을 위해 화면 낭독기가 읽어줄 설명입니다. (필수 예의!)
  • width/height 속성: 이미지의 너비/높이를 조절합니다. (보통은 나중에 CSS로 조절합니다)
<img src="cat.jpg" alt="귀여운 고양이 사진" width="300">

👨‍💻 링크와 이미지 합체!

이미지를 클릭했을 때 다른 페이지로 넘어가게 만들 수도 있습니다. <a> 태그 사이에 글자 대신 <img> 태그를 넣으면 됩니다!

⚡ 미디어 태그 테스트

🎯 속성 확인 미션

<img> 태그에서 시각 장애인을 위한 화면 낭독기가 읽어주거나, 엑스박스(이미지 깨짐) 발생 시 대신 나타나는 텍스트를 적는 속성은 무엇일까요?

1. src="설명"
2. href="설명"
3. alt="설명"
4. title="설명"

📌 핵심 요약

  • 속성(Attribute): 태그에 추가 정보를 제공합니다. (예: href, src)
  • <a href="주소">: 클릭 시 이동할 링크를 만듭니다. (target="_blank"로 새창 열기 가능)
  • <img src="주소" alt="설명">: 화면에 이미지를 표시합니다. 닫는 태그가 없습니다.
  • alt 속성: 접근성을 위해 이미지 설명을 반드시 적어주는 것이 좋습니다.

📊 6. 표(Table) 만들기

데이터를 깔끔하게 행과 열로 정리해봐요!

1️⃣ 표를 구성하는 4총사

표(Table)를 만들 때는 엑셀처럼 여러 겹의 상자가 필요합니다.

🏢 아파트 짓기에 비유해 볼까요?
- <table>: 아파트 건물 전체
- <tr>: 아파트의 각 층수 (Table Row, 가로줄)
- <th>, <td>: 각 층에 있는 호실들 (칸). th는 진한 제목칸(Header), td는 일반 데이터칸(Data)입니다. 
<table border="1">
  <!-- 첫 번째 줄 (제목) -->
  <tr>
    <th>이름</th>
    <th>나이</th>
  </tr>
  <!-- 두 번째 줄 (데이터) -->
  <tr>
    <td>홍길동</td>
    <td>20</td>
  </tr>
</table>

2️⃣ 칸 합치기 (병합)

가끔 엑셀에서 셀을 병합하는 것처럼 표의 칸을 합쳐야 할 때가 있죠?

  • colspan: 가로(열, Column)로 칸을 합칩니다.
  • rowspan: 세로(행, Row)로 칸을 합칩니다.
⚡ 표 병합 테스트

🎯 표 만들기 미션

표를 만들 때 '가로 한 줄(행)'을 통째로 묶어주는 태그는 무엇일까요?

1. <table>
2. <tr>
3. <td>
4. <th>

📝 7. 입력 폼(Form) 기초

사용자에게 데이터를 입력받아 볼까요? (회원가입, 로그인)

1️⃣ 폼의 껍데기: <form> 태그

회원가입 버튼을 누르면 입력한 정보가 서버로 날아갑니다. 이 정보들을 하나로 묶어주는 거대한 우체통이 바로 <form> 태그입니다.

<form action="/login" method="POST">
  <!-- 여기에 입력창과 버튼들이 들어갑니다 -->
</form>

2️⃣ 다재다능한 카멜레온: <input> 태그

<input> 태그는 type 속성에 무엇을 쓰느냐에 따라 텍스트창, 비밀번호창, 체크박스 등 수십 가지 모양으로 변신합니다!

⚡ 입력 폼 변신 테스트

🎯 폼 미션

비밀번호를 입력할 때 글자가 까만 동그라미(●●●)로 가려지게 하려면 <input> 태그의 type 속성을 무엇으로 해야 할까요?

🏗️ 9. 시맨틱(Semantic) 태그

이름표가 달린 투명 상자들로 웹페이지를 깔끔하게!

1️⃣ 태그에도 의미(Semantic)가 있다!

과거에는 구역을 나눌 때 오직 <div> (투명 상자) 하나만 무수히 많이 사용했습니다. 하지만 이러면 컴퓨터(검색 엔진)가 어디가 중요한 본문이고 어디가 메뉴인지 알 수 없었죠.

🏷️ <div>에 이름표를 붙여준 시맨틱 태그
- <header>: 사이트의 머리말 (로고, 상단바)
- <nav>: 네비게이션 메뉴 (메뉴판)
- <main>: 가장 중요한 본문 내용
- <footer>: 사이트 꼬리말 (회사 주소, 저작권)

이 태그들은 화면에 그릴 때는 <div>와 똑같은 투명 상자지만, 시각장애인을 위한 화면 낭독기나 구글 검색 로봇에게 "여기가 메뉴야!"라고 친절하게 알려주는 역할을 합니다.

📌 핵심 요약

  • 시맨틱 태그: 단순히 구역을 나누는 것을 넘어, 그 구역의 '의미'를 컴퓨터에게 알려주는 태그들입니다.
  • <div>와 <span>: 특별한 의미 없이 구역을 나눌 때 쓰는 투명 상자입니다. div는 줄바꿈을 하고, span은 줄바꿈을 하지 않습니다.

🎨 11. CSS 시작하기

밋밋한 HTML에 화려한 색상과 디자인을 입혀보자!

1️⃣ CSS란 무엇일까?

CSS (Cascading Style Sheets)는 HTML로 만들어진 뼈대 위에 예쁜 옷을 입히는 기술입니다. 색상, 크기, 위치, 폰트 등을 모두 CSS가 결정합니다.

👗 누가 입을지, 어떤 옷을 입을지 지정하기
CSS는 기본적으로 "누구한테(선택자), 어떤 디자인을(속성), 어떻게(값) 적용할래?" 라는 규칙으로 작성됩니다. 
/* 선택자 { 속성: 값; } */
h1 {
  color: red;        /* 글자색을 빨간색으로 */
  font-size: 24px;   /* 글자 크기를 24픽셀로 */
}

2️⃣ CSS를 적용하는 3가지 방법

① 인라인 (Inline) 스타일

HTML 태그 안에 직접 style="..." 속성을 적어 넣는 방법입니다. (간단하지만 코드가 지저분해져서 비추천!)

<h1 style="color: blue;">파란색 제목</h1>

② 내부 스타일 시트 (Internal)

HTML 문서의 <head> 태그 안에 <style> 태그를 만들고 그 안에 몰아서 적는 방법입니다.

<head>
  <style>
    h1 { color: blue; }
  </style>
</head>

③ 외부 스타일 시트 (External) ⭐가장 추천⭐

CSS 코드를 style.css 같은 별도의 파일로 빼내고, HTML에서 <link> 태그로 불러오는 방법입니다. 디자인과 뼈대를 분리해서 관리하기 아주 좋습니다!

<head>
  <!-- CSS 파일을 연결합니다! -->
  <link rel="stylesheet" href="style.css">
</head>

🎯 12. CSS 선택자 (Selector)

디자인을 적용할 대상을 정확하게 콕! 집어내기

1️⃣ 클래스(Class)와 아이디(ID) 선택자

HTML 태그에 별명(Class)이나 고유번호(ID)를 붙여서 특정 요소만 디자인할 수 있습니다.

🏷️ Class는 동아리 명찰, ID는 주민등록번호!
- Class (.점): 여러 명에게 같은 명찰을 줄 수 있습니다. (재사용 O)
- ID (#샵): 한 문서 안에서 오직 한 명만 가질 수 있습니다. (재사용 X) 
/* HTML: <div class="box"></div> */
.box { 
  color: red;  /* 점(.)으로 시작하면 클래스! */
}

/* HTML: <h1 id="title"></h1> */
#title { 
  color: blue; /* 우물정(#)으로 시작하면 아이디! */
}

📦 13. 박스 모델 (Box Model)

모든 HTML 요소는 사실 네모난 박스입니다!

1️⃣ 마진, 패딩, 보더의 차이

웹디자인에서 가장 헷갈리고 중요한 개념입니다! 모든 요소는 내용물을 감싸는 투명한 박스들을 겹겹이 입고 있습니다.

🖼️ 액자에 비유해 볼까요?
- Content (내용): 실제 사진이나 그림
- Padding (패딩): 사진과 액자 틀 사이의 여백 (안쪽 여백)
- Border (보더): 액자의 나무 틀 (테두리 두께)
- Margin (마진): 액자와 내 방 벽 사이의 띄워진 거리 (바깥 여백) 
.box {
  width: 200px;           /* 내용물의 너비 */
  padding: 20px;          /* 안쪽 여백: 내용물이 벽에 안 붙게 띄움 */
  border: 5px solid black;/* 테두리: 5px 두께의 검은색 실선 */
  margin: 30px;           /* 바깥 여백: 다른 박스와 거리 두기 */
}

👨‍💻 마진/패딩 실시간 테스트

⚡ 박스 모델 조작하기

📌 핵심 요약

  • margin: 박스의 '바깥쪽' 여백 (남과 거리두기)
  • padding: 박스의 '안쪽' 여백 (내 살 찌우기)
  • border: 박스의 '테두리' (경계선 그리기)
  • box-sizing: border-box: 패딩이나 테두리를 주면 박스 전체 크기가 커지는 현상을 막아주는 마법의 주문입니다.

📐 15. Flexbox 레이아웃

박스들을 가로로 나란히, 가운데 정렬하기의 끝판왕!

1️⃣ display: flex의 마법

과거에는 박스를 가로로 나란히 배치하거나 정가운데에 놓기 위해 엄청난 노가다를 했습니다. 하지만 이제 flex 하나면 모든 것이 해결됩니다!

📦 부모 박스에게 주문 걸기
Flexbox는 자식 박스에게 명령하는 것이 아니라, 자식들을 감싸고 있는 부모 상자(Container)에게 마법을 겁니다.
"부모 상자야! 너의 자식들을 가로로 쭉 정렬해!" 
.container {
  display: flex;            /* 마법 시작! (기본적으로 자식들이 가로로 배치됨) */
  justify-content: center;  /* 가로 기준 중앙 정렬 */
  align-items: center;      /* 세로 기준 중앙 정렬 */
}

📐 16. Grid 레이아웃

바둑판 모양의 2차원 레이아웃을 완벽하게 통제하기

1️⃣ Flex vs Grid 차이점

Flexbox가 '한 줄 기차(1차원)'를 정렬하는 데 특화되어 있다면, Grid는 엑셀이나 체스판처럼 '가로세로 격자(2차원)' 레이아웃을 잡는 데 사용합니다. 갤러리나 쇼핑몰 상품 목록을 만들 때 필수적입니다.

.grid-container {
  display: grid;
  /* 기둥(Column) 3개를 같은 비율(1fr)로 만들어라! */
  grid-template-columns: 1fr 1fr 1fr; 
  gap: 20px; /* 박스 사이의 간격 */
}

👨‍💻 쇼핑몰 상품 목록 만들기

⚡ Grid 갤러리 테스트

📌 CSS 레이아웃 핵심 요약

  • display: flex: 가로 또는 세로 '한 방향'으로 박스를 나열하고 정렬할 때 사용. (네비게이션바, 버튼 그룹)
  • display: grid: 가로세로 '양방향' 바둑판 모양으로 영역을 나눌 때 사용. (쇼핑몰, 사진 갤러리)
  • justify-content: 메인 축(가로)의 정렬 방식을 결정합니다. (center, space-between 등)
  • gap: Flex나 Grid 안에서 자식 박스들 사이의 여백(간격)을 띄워줍니다. (margin보다 훨씬 편리!)

1️⃣ JavaScript란?

버튼을 눌렀을 때 팝업창이 뜨거나, 사진이 옆으로 스르륵 넘어가는 등 웹페이지의 모든 '동작'을 담당하는 프로그래밍 언어입니다.

<!-- 자바스크립트는 <script> 태그 안에 작성합니다 -->
<script>
  // 이건 자바스크립트 주석입니다.
  alert("환영합니다!"); // 화면에 경고창(팝업) 띄우기
</script>

2️⃣ 변수(Variable) 만들기

데이터를 담아두는 상자입니다. 옛날에는 var를 썼지만, 요즘은 let과 const만 사용합니다!

📦 상자의 종류
- let: 내용물을 언제든지 바꿀 수 있는 재사용 상자
- const: 한 번 넣으면 절대 바꿀 수 없는 자물쇠 상자 (Constant, 상수) 
<script>
  let age = 20;
  age = 21; // 변경 가능! (에러 안 남)

  const name = "홍길동";
  // name = "임꺽정"; // 에러 발생! (const는 변경 불가)
</script>

3️⃣ 개발자의 친구: console.log

코드가 잘 돌고 있는지, 변수에 무슨 값이 들어있는지 확인할 때 화면에 팝업을 띄우면 사용자가 놀라겠죠? 그래서 개발자들만 볼 수 있는 비밀 노트(콘솔창)에 글을 적습니다.

<script>
  let result = 10 + 20;
  console.log("계산 결과는: " + result);
  // 브라우저에서 F12를 눌러 [Console] 탭을 열면 볼 수 있어요!
</script>

🎯 18. DOM 조작 기초

자바스크립트로 HTML 문서(DOM)를 마음대로 조종하기

1️⃣ DOM (Document Object Model)

자바스크립트가 HTML 태그들을 쉽게 찾고 조작할 수 있도록, 브라우저가 HTML 문서를 나무 구조(Tree)의 객체로 만들어 놓은 것입니다.

🕵️‍♂️ 특정 태그를 찾는 탐정 명렁어
"문서(document)야! 아이디가 'title'인 녀석 좀 찾아와!"
👉 document.getElementById("title")

2️⃣ 글자와 스타일 바꾸기

요소를 찾았으면 내용(innerHTML)이나 스타일(style)을 마음대로 바꿀 수 있습니다.

<!-- HTML -->
<h1 id="greeting">안녕하세요</h1>

<script>
  // 1. 요소 찾기
  let title = document.getElementById("greeting");
  
  // 2. 내용 바꾸기
  title.innerHTML = "반갑습니다!";
  
  // 3. 스타일 바꾸기 (CSS와 다르게 카멜표기법 사용)
  title.style.color = "blue";
  title.style.fontSize = "50px"; // font-size가 아님!
</script>

🖱️ 19. 이벤트 핸들링 (Event)

클릭! 타이핑! 마우스 이동! 사용자의 행동에 반응하기

1️⃣ 이벤트(Event)란?

사용자가 버튼을 '클릭'하거나, 키보드를 '입력'하는 모든 행동을 이벤트라고 합니다. 우리는 특정 이벤트가 발생할 때까지 기다렸다가(Listen) 반응(Handle)하게 만들 수 있습니다.

🎧 귀를 쫑긋 세우고 기다리기: addEventListener
"버튼아, 누가 너를 '클릭(click)'하는지 귀 기울여 듣고 있다가, 클릭하면 이 함수를 실행해!" 
<!-- HTML -->
<button id="myBtn">나를 눌러봐!</button>

<script>
  const btn = document.getElementById("myBtn");
  
  // 'click' 이벤트를 들으면, 팝업 띄우기 함수 실행!
  btn.addEventListener("click", function() {
    alert("버튼이 클릭되었습니다!");
  });
</script>

  • document.getElementById("아이디"): HTML 문서를 조종하기 위해 특정 태그를 찾는 가장 흔한 방법입니다.
  • .innerHTML: 태그 안의 내용물(텍스트나 HTML 요소)을 통째로 가져오거나 바꿉니다.
  • .addEventListener("이벤트", 함수): 클릭(click), 마우스 오버 등 특정 상황이 발생할 때 함수를 실행시킵니다.
