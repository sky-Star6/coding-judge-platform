💡 Chapter 1. LangGraph 기초 & 개발 환경 구축

기존의 챗봇은 사용자가 질문하면 한 번 대답하고 끝나는 '일직선' 구조였습니다. 하지만 LangGraph는 다릅니다!

💡 눈높이 비유: "집요한 천재 요리사"

LangGraph는 일직선 컨베이어 벨트가 아니라, 요리가 완벽해질 때까지 간을 보고 다시 끓이는 '순환 구조'를 가진 요리사와 같습니다. 과정 중에서 만족스럽지 않으면 언제든 이전 단계로 다시 돌아갈 수 있죠!

🔑 3대 핵심 키워드 정리
  • Node (노드): 특정 작업을 수행하는 파이썬 함수. (예: "검색해", "요약해", "검토해" 등 행동 담당)
  • Edge (엣지): 노드 사이의 이동 경로. 다음으로 넘어갈지, 전으로 돌아갈지 정하는 다리 역할.
  • State (상태): 모든 과정을 기록하는 '공용 수첩(Dictionary)'. 작업 결과가 여기에 누적됩니다.
🛠️ 개발 환경 구축: "무균 실험실 만들기"

파이썬 패키지를 컴퓨터 전체에 그냥 설치하면 나중에 버전이 꼬여서 대참사가 발생합니다. 폴더 안에 나만의 조그만 무균 실험실(가상환경, venv)을 만들어 주고, 그 안에서만 LangGraph를 설치합니다.

# 1. 가상환경 생성 (이름: venv)
python -m venv venv

# 2. 가상환경 활성화 (윈도우 기준)
venv\Scripts\activate

# 3. pip 업그레이드 및 필수 라이브러리 설치
python -m pip install --upgrade pip
pip install langgraph

📌 주의사항: LangGraph는 Python 3.10 이상 (추천 3.12) 환경을 권장합니다.

💻 실습 코드 & 상세 분석: 가장 기초적인 형태
from typing import TypedDict
from langgraph.graph import StateGraph

# 1. State (상태) 정의: 우리가 기억할 데이터 종류를 정합니다.
class State(TypedDict):
    num: int

# 2. Node (노드) 만들기: 실행할 함수들입니다.
def step1(state):
    print("Step 1 실행")
    # State의 'num'에 10을 덮어씌웁니다.
    return {"num": 10}

def step2(state):
    print("Step 2 실행")
    print("받은 값:", state["num"]) # 앞서 저장된 10을 가져와서 사용
    return {}

# 3. 그래프 연결 (조립 과정)
builder = StateGraph(State)   # 뼈대 준비
builder.add_node("step1", step1) # 노드 부착
builder.add_node("step2", step2) 
builder.set_entry_point("step1") # 시작점은 step1
builder.add_edge("step1", "step2") # step1이 끝나면 무조건 step2로 가라!

# 4. 컴파일 & 실행
graph = builder.compile()   # 로봇 조립 완료
graph.invoke({})            # 빈 상태({})로 시작 버튼(invoke) 누르기!

🔀 Chapter 3. 흐름 분기와 갈림길 (조건문)

무조건 정해진 순서대로만(add_edge) 다음 단계로 이동한다면 똑똑한 판단을 내리는 AI라고 할 수 없겠죠? 이번 챕터에서는 조건(If)에 따라 갈림길을 만드는 방법을 배웁니다.

💡 눈높이 비유: "스마트 표지판"

단순 화살표가 "닥치고 다음으로 가!" 라면, 조건부 화살표(add_conditional_edges)는 "지금 상태(점수, 기분, 날씨 등)를 평가하고 판단해서 최적의 방향 골라가!"라는 스마트 표지판 역할을 합니다.

🔑 필수 키워드
  • add_conditional_edges: 특정 노드가 끝났을 때, 판단 라우터 함수의 결과에 따라 분기(갈림길)를 생성하는 함수입니다.
  • END: LangGraph에서 "여기가 끝이야. 더 이상 엣지를 타지 마"라고 선언하는 특별한 내장 변수입니다.
💻 실습 코드: 50점이 넘으면 멈춰라!
from typing import TypedDict
from langgraph.graph import StateGraph, END

class State(TypedDict):
    num: int

# [노드] 무조건 숫자에 10을 더하는 작업체
def add_ten(state: State):
    current = state.get("num", 0)
    print(f"현재 숫자: {current} -> 10을 더합니다.")
    return {"num": current + 10}

# [조건 함수 (스마트 표지판)] 갈림길의 방향을 지시해주는 함수
def should_continue(state: State):
    if state["num"] >= 50:
        return "end"     # 50 이상이면 "end"라는 이름표 뱉기
    else:
        return "continue" # 아닐 때는 "continue" 이름표 뱉기

builder = StateGraph(State)
builder.add_node("add_ten", add_ten)
builder.set_entry_point("add_ten")

# [결정적 순간] 스마트 표지판 부착!
builder.add_conditional_edges(
    "add_ten",           # 1. 출발지: add_ten 작업이 끝나면
    should_continue,     # 2. 어떤 기준으로? should_continue 물어봐라!
    {
        "end": END,                # 이름표가 "end"면 완전 프로그램 종료.
        "continue": "add_ten"      # 이름표가 "continue"면 다시 "add_ten" 노드로 회귀!
    }
)
app = builder.compile()

# 실행: 시작 입력값을 0으로 줘볼까요?
app.invoke({"num": 0})
💡 코드 핵심 분석: state.get("num", 0) 이란?

일반적인 파이썬 문법인 state["num"]을 사용하면, 아직 데이터가 생성되지 않은 맨 처음 실행 시 존재하지 않는 방을 찾으려 속출하는 KeyError가 발생하여 봇이 뻗어버립니다.

하지만 딕셔너리의 안전한 호출 함수인 .get("num", 0)을 사용하면,
"num 이라는 값이 있으면 그 값을 쓰고, 만약 아예 비어있다면 에러를 내지 말고 조용히 '0'을 꺼내와!"라는 뜻이 됩니다. 끊김 없이 순환해야 하는 LangGraph 생태계에서 에러를 방지하는 필수 테크닉입니다.

🔄 Chapter 4. 반복문 제어 (루프)

파이썬의 선형적인 while, for문과 달리 LangGraph의 반복문은 엣지(Edge)를 이전 노드로 다시 화살표를 이어붙이는 '사이클' 형식으로 구현합니다.

💡 눈높이 비유: "합격 통지서 vs 스탬프 랠리"
  • While 방식 (조건 기반): 오디션 합격증을 받을 때까지 계속 실기실로 돌아가는 것입니다. (결과의 質이 목표)
  • For 방식 (횟수 기반): 결과가 어떻든 도장을 5번 꽉 채우면 무조건 집으로 돌아가는 것입니다. (안전 장치 목표)
💻 실습 코드: 정해진 횟수만큼 돌리기 (For 방식 유사 구현)

파이썬 언어 자체의 for 루프가 아니라, State 안에 직접 카운터(count)를 저장하고 늘려가며 제어해야 합니다.

import random
from typing import TypedDict
from langgraph.graph import StateGraph, END

class State(TypedDict):
    count: int      # 반복 횟수를 누적하는 변수
    max_loops: int  # 끝내기 위한 목표 횟수 변수

# 노드: 실행될 때마다 카운트를 +1 합니다.
def counter_node(state: State):
    current_count = state.get("count", 0)
    print(f"현재 횟수: {current_count + 1} / {state['max_loops']}")
    return {"count": current_count + 1}  # State에 새로운 횟수를 저장!

# 라우터 함수: 횟수에 도달했는지 검사
def check_count(state: State):
    if state["count"] < state["max_loops"]:
        return "loop"
    else:
        print("✅ 목표 횟수 도달. 스탬프 완성!")
        return "end"

builder = StateGraph(State)
builder.add_node("counter_node", counter_node)
builder.set_entry_point("counter_node")

# 반복 판단 연결 다리
builder.add_conditional_edges(
    "counter_node", 
    check_count, 
    {"loop": "counter_node", "end": END}
)

app = builder.compile()
if __name__ == "__main__":
    app.invoke({"count": 0, "max_loops": 3})

🏆 Chapter 5. 종합 실전 챌린지

지금까지 배운 모든 State, 조건문, 분기, 루프 요소를 겹치게 하여 혼자서 해결해 볼 수 있는 연습 미션 세트입니다!

📝 종합 미션: "동물 나라의 암호를 맞춰라! (While 무한루프 + 분기)"

사용자로부터 단어를 입력받는 노드 구조입니다. 조건은 아래와 같습니다.

  1. 사용자의 입력(`state["text"]`)이 "열려라참깨"라면 즉시 `treasure_node` 방향으로 보냅니다.
  2. 단어가 "강아지" 거나 "고양이" 라면 `animal_node` 라는 다른 방향으로 보냅니다.
  3. 그 외의 단어라면 "틀렸습니다!"를 출력하고 `entrance_node`(다시 처음 입구)로 강제로 반환시킵니다.

위 3가지 논리를 `elif` 와 `or` 를 복합적으로 사용하여 거대한 뼈대 코드를 조립해 보세요!

정답 구조 확인하기 
def password_router(state: State):

        return "go_to_treasure"
    elif text == "강아지" or text == "고양이":
        return "go_to_animal"
    else:
        print("틀렸습니다! 다시 시작!")
        return "retry_entrance"
