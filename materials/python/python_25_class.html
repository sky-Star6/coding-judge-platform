1️⃣ 클래스(Class)란? 설계도와 제품 📐

비슷한 모양과 기능을 가진 데이터(변수)와 행동(함수)을 하나로 묶어놓은 설계도(틀)입니다.

🍞 마스터 코치의 비유: 붕어빵 틀
• 클래스(Class): 붕어빵을 찍어내는 틀(설계도)입니다.
• 객체(Object/Instance): 틀에서 만들어져 나온 진짜 붕어빵(제품)입니다.
틀 하나만 있으면 팥 붕어빵, 슈크림 붕어빵을 계속 만들 수 있죠!

# 1. 클래스(틀) 만들기
class FishBread:
    pass  # 일단 빈 틀 생성 (내용 없음)

# 2. 객체(붕어빵) 찍어내기
bread1 = FishBread()  # 첫 번째 붕어빵 굽기!
bread2 = FishBread()  # 두 번째 붕어빵 굽기!

print(type(bread1)) # <class '__main__.FishBread'>

2️⃣ 클래스 생성자 `__init__`: 붕어빵 굽기 시작! 🔥

객체(instance)가 처음 생성될 때 자동으로 실행되는 특별한 함수입니다. (초기 설정 담당)

🥣 마스터 코치의 비유: 앙금 넣기
틀에 반죽을 부을 때, "이 붕어빵은 어떤 맛으로 구울까?" 결정하는 초기 작업입니다.
이름 양옆에 언더스코어 두 개(`__`)가 붙은 것은 파이썬의 마법 메서드(Magic Method)입니다. 
class FishBread:
    # 생성자: 붕어빵이 구워질 때 가장 먼저 호출됨!
    def __init__(self, flavor, price):
        self.flavor = flavor  # 내 앙금은 이거야!
        self.price = price    # 내 가격은 이거야!

# 붕어빵을 구우면서 동시에 맛과 가격을 정해줌
redbean = FishBread("팥", 1000)
cream = FishBread("슈크림", 1200)

print(redbean.flavor)  # 팥
print(cream.price)     # 1200

3️⃣ 클래스에서 `self`의 의미: "나는 나!" 🙋‍♂️

클래스 안의 모든 함수(메서드)는 첫 번째 재료(매개변수)로 무조건 self를 받아야 합니다.

🙋‍♂️ 마스터 코치의 비유: 자기 자신 가리키기
공장(클래스)에서 여러 개의 붕어빵(객체)을 만들었습니다.
"가격 올려!" 라고 명령했을 때, 어떤 붕어빵의 가격을 올릴지 어떻게 알까요?
self는 "바로 나(자신)의 데이터를 바꿔라!"라고 콕 집어주는 역할을 합니다. 
class FishBread:
    def __init__(self, flavor, price):
        self.flavor = flavor
        self.price = price

    # 가격을 올리는 함수 (첫 번째 매개변수는 무조건 self)
    def raise_price(self, amount):
        # '내(self)' 가격을 amount만큼 올린다!
        self.price = self.price + amount
        print(f"{self.flavor} 붕어빵 가격이 {self.price}원으로 올랐습니다!")

# 슈크림 붕어빵 구워서 가격 300원 더 올리기
cream = FishBread("슈크림", 1200)
cream.raise_price(300) # 파이썬이 알아서 self에 'cream'을 넣어줍니다!
# 출력: 슈크림 붕어빵 가격이 1500원으로 올랐습니다!

🚨 중요: cream.raise_price(300) 이라고 호출해도, 파이썬은 몰래 raise_price(cream, 300) 처럼 self 자리에 cream 객체 자신을 던져줍니다.

4️⃣ 📝 설계사 자격 시험

1. 다음 설명 중 틀린 것은 무엇일까요?
  • a. 클래스(Class)는 객체를 만들기 위한 틀이다.
  • b. 생성자 `__init__`은 객체가 만들어질 때 자동으로 호출된다.
  • c. 클래스 내부의 함수(메서드)는 `self`를 안 써도 된다.
  • d. `self`는 만들어진 객체 자신을 가리킨다.

📌 핵심 요약

  • 클래스(Class): 똑같은 모양의 객체를 계속 찍어낼 수 있는 틀(설계도).
  • 객체(Instance): 클래스라는 틀에서 만들어진 실제 결과물.
  • `__init__` (생성자): 객체가 처음 생성될 때 실행되는 초기화 함수.
  • `self`: 찍혀 나온 무수한 객체 중 '자기 자신'을 콕 집어 가리키는 대명사.

이제 여러분만의 세상을 창조할 수 있는 설계도를 얻었습니다! 💠
