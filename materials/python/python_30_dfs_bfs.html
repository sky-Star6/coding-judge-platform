🌌 파이썬 마스터 코치: 30. DFS & BFS

[논리적 사고 II] 관계를 탐험하는 두 가지 방법: '미로 탈출과 인맥 지도'

1️⃣ 그래프 기초: 노드(Node)와 간선(Edge)으로 세상 표현하기 🕸️

세상의 많은 문제들은 "무엇이 서로 연결되어 있는가?"로 표현할 수 있습니다. 이것을 컴퓨터에게 이해시키기 위해 만든 그림을 그래프(Graph)라고 부릅니다.

🗺️ 마스터 코치의 비유: 지하철 노선도
• 노드(Node/정점): 강남역, 역삼역 같은 특정 '장소'나 '사람'입니다.
• 간선(Edge/링크): 두 역을 이어주는 '철로'나 사람 사이의 '관계'입니다.
이런 구조를 파이썬에서는 주로 딕셔너리(Dictionary)를 사용해 표현합니다. 
# 그래프를 컴퓨터가 이해하도록 딕셔너리로 만들기
# '나'가 '영희'와 '철수'를 알고 있다는 뜻으로 표현합니다.
graph = {
    '나': ['영희', '철수'],
    '영희': ['나', '민수'],
    '철수': ['나'],
    '민수': ['영희']
}

2️⃣ 준비물 체크: 스택(Stack)과 큐(Queue) 자료구조 다시 보기 🎒

지도를 들고 탐험을 떠나기 전, 메모를 남길 핵심 도구 두 가지를 꼭 챙겨야 합니다.

🥞 1. 스택(Stack) = "팬케이크(후입선출, LIFO)"
나중에 구운 팬케이크를 가장 먼저 위에서 집어 먹습니다.
파이썬에서는 일반 리스트(`append`, `pop`)를 그대로 스택으로 사용합니다.
🚶‍♂️ 2. 큐(Queue) = "맛집 줄서기(선입선출, FIFO)"
먼저 줄을 선 사람이 먼저 식당에 들어갑니다.
파이썬에서는 리스트 대신 `collections.deque` 모듈을 가져와서 사용해야 엄청 빠릅니다! (`append`, `popleft`)

3️⃣ 깊이 우선 탐색(DFS): 한 우물만 파는 끈기 있는 탐색 ⛏️

DFS(Depth-First Search)는 하나의 경로를 따라 갈 수 있을 때까지 가장 깊게(끝까지) 계속 들어가는 방식입니다.

🔦 마스터 코치의 비유: 미로 탈출
미로에 갇혔을 때, 갈림길이 나오면 일단 한쪽 길만 골라 막다른 길이 나올 때까지 직진합니다.
벽에 부딪히면? 방금 지나친 가장 가까운 갈림길로 다시 돌아와서(이 과정을 백트래킹, Backtracking이라 합니다) 다른 길을 갑니다.
돌아올 곳을 기억해야 하므로 스택(Stack)이나 재귀 함수를 사용합니다. 
def dfs(graph, start):
    visited = []            # 내가 지나온 발자국 기록
    stack = [start]         # 스택에 첫 시작점을 넣고 시작! (갈 곳 목록)
    
    while stack:
        node = stack.pop()  # 스택 맨 위(가장 최근에 발견한 길)에서 꺼냄
        if node not in visited:
            visited.append(node)         # 발자국 꽝!
            # 아직 방문안한 이웃 노드들을 스택에 쓸어 담음
            # (끝까지 가기 위해 새로 발견한 길을 맨 위에 올림)
            stack.extend(graph[node])
            
    return visited

4️⃣ 너비 우선 탐색(BFS): 가까운 곳부터 살피는 최단 거리 탐색 📡

BFS(Breadth-First Search)는 시작점을 중심으로 마치 물수제비 파동이 퍼지듯 가까운 주변부터 샅샅이 탐색하는 방식입니다.

🤝 마스터 코치의 비유: 인맥 지도 거리 찾기 (케빈 베이컨 게임)
"내 친구의 친구 중에 의사가 있을까?"
먼저 나의 직속 친구(1단계)를 모두 살펴봅니다. 의사가 없다면, 그 다음엔 친구들의 친구(2단계)를 전부 살펴봅니다.
이렇게 가까운 1촌, 2촌, 3촌 순서대로 넓혀가기 때문에, 의사를 발견한다면 그 경로가 무조건 "나와 가장 가까운(최단 거리)" 인맥이 됩니다.
순서대로 처리해야 하므로 큐(Queue)를 사용합니다. 
from collections import deque  # 파이썬에서 큐를 쓸 때 필수!

def bfs(graph, start):
    visited = []             # 지나온 발자국 기록
    queue = deque([start])   # 큐에 첫 시작점을 줄 세우기!
    
    while queue:
        node = queue.popleft()  # 큐의 맨 앞(가장 먼저 줄 선) 노드를 꺼냄
        if node not in visited:
            visited.append(node)            # 방문 기록!
            # 인접한 노드들을 새로 발견한 순서대로 뒤에 줄을 세움
            queue.extend(graph[node])
            
    return visited

5️⃣ 비교 분석: 어떤 상황에서 어떤 탐색을 써야 할까? ⚖️

문제의 결에 따라 적절한 도구를 선택하는 것이 고수의 실력입니다!

항목 DFS (깊이 우선) 🔦 BFS (너비 우선) 📡
사용 도구 스택(Stack) 또는 재귀 함수 큐(Queue, deque)
탐색 느낌 한 놈만 팬다! 막히면 돌아온다. 나를 중심으로 주변부터 둥글게 둥글게 퍼져나간다.
장점 경로의 "특징"을 저장해야 할 때 좋음.
(예: 지나온 길에 특정 장애물이 있으면 안 됨)		 "최단 거리" 최적해를 찾는 문제에 무조건 유리함.
(예: 출발지에서 목적지까지 가장 빨리 가는 길)
단점 운이 나쁘면 정답(출구)이 바로 옆인데, 이상한 깊은 곳에 먼저 빠져서 한참 헤맬 수 있음. 넓게 퍼지면서 모두 기억해야 하므로, 메모리(저장 공간)를 많이 먹음.

📌 핵심 요약

  • 그래프(Graph): 세상의 연결점을 노드(Node)와 간선(Edge)으로 표현한 지도.
  • DFS (깊이 우선 탐색): 끝을 볼 때까지 직진! 막히면 되돌아오는 끈기의 미로 탐험가. (스택 사용)
  • BFS (너비 우선 탐색): 주변부터 천천히 모든 가능성을 훑는 레이더망! 나의 1촌부터 찾는 인맥 지도. (큐 사용)
  • 꿀팁: "모든 경로를 방문"해야 한다면 DFS, "최단 거리"를 찾아야 한다면 BFS를 떠올리세요!
