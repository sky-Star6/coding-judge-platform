🧭 파이썬 마스터 코치: 29. 이진 탐색

[논리적 사고 I] 데이터가 많을수록 빛나는 보물 찾기 알고리즘!

1️⃣ 전제 조건: 왜 데이터가 정렬(Sort)되어 있어야 할까? 📚

이진 탐색(Binary Search)은 아무 데서나 쓸 수 없습니다. 반드시 데이터가 순서대로 정렬(Sorted)되어 있어야만 마법이 발동합니다.

📖 마스터 코치의 비유: 두꺼운 국어사전
사전에서 '파이썬'이라는 단어를 찾을 때, 첫 장부터 한 장씩 넘기는 사람은 없죠?
사전은 알파벳/가나다 순으로 이미 정렬되어 있기 때문에, 대충 중간을 펼치고 앞인지 뒤인지 판단할 수 있는 것입니다!

2️⃣ 분할 정복의 원리: 범위를 절반씩 날려버리는 마법 ⚔️

정중앙을 찔러보고, 내가 찾는 값이 중앙값보다 큰지 작은지 비교해서 나머지 절반을 아예 버리는(검색에서 제외하는) 방식입니다.

📈 마스터 코치의 비유: 업 & 다운 게임 (Up & Down)
친구: "1부터 100 사이의 숫자를 맞춰봐!"
나: "50!"
친구: "UP!" (아, 그럼 1~50은 다 버리고 51~100 사이에서 중앙인 75를 부르면 되겠군!)

이처럼 검색해야 할 범위가 100 -> 50 -> 25 -> 12... 식으로 절반씩 숭텅숭텅 날아가는 마법이 파라독스처럼 보일 정도로 엄청난 속도를 냅니다.

3️⃣ Python 실습: 반복문(While)과 재귀(Recursion)로 구현하기 💻

이진 탐색은 가장 기본적으로 while 반복문을 쓰거나, 스스로를 다시 부르는 재귀 함수를 써서 구현합니다.

🔹 반복문(While) 방식 (가장 많이 씀)

def binary_search_while(arr, target):
    left = 0                  # 탐색 범위의 맨 왼쪽 끝 (시작점)
    right = len(arr) - 1        # 탐색 범위의 맨 오른쪽 끝 (끝점)

    while left <= right:        # 범위가 엇갈릴 때까지 반복
        mid = (left + right) // 2 # 정중앙 찌르기!

        if arr[mid] == target:
            return mid          # 보물 발견! (인덱스 반환)
        elif arr[mid] < target:
            left = mid + 1      # 타겟이 더 크니까 오른쪽 절반만 살리자
        else:
            right = mid - 1     # 타겟이 더 작으니까 왼쪽 절반만 살리자

    return None                 # 다 찾았는데 없으면 None

data = [5, 12, 24, 35, 42, 55, 68, 72] # 🚨 정렬되어 있어야 함
print(binary_search_while(data, 35)) # 3 출력 (인덱스 3번째 있음)

🔹 재귀(Recursion) 방식

def binary_search_recursive(arr, target, left, right):
    if left > right:
        return None

    mid = (left + right) // 2

    if arr[mid] == target:
        return mid
    elif arr[mid] < target:
        # 자기 자신을 다시 부르면서 범위를 (mid+1 ~ right)로 좁힘
        return binary_search_recursive(arr, target, mid + 1, right)
    else:
        # 자기 자신을 다시 부르면서 범위를 (left ~ mid-1)로 좁힘
        return binary_search_recursive(arr, target, left, mid - 1)

4️⃣ 성능 분석: $O(N)$과 $O(\log N)$의 압도적인 차이 🚀

코딩 테스트에서 이진 탐색이 필수인 이유는 데이터가 폭발적으로 많아질 때 그 진가가 드러나기 때문입니다.

데이터 개수 (N) 순차 탐색 ($O(N)$) 최대 시도 횟수 이진 탐색 ($O(\log N)$) 최대 시도 횟수
100 개 100 번 7 번
10,000 개 10,000 번 14 번
1,000,000 개 (100만) 1,000,000 번 20 번

100만 개의 데이터 속에서 보물을 찾는데 단 20번의 비교만으로 끝을 냅니다. 순차 탐색(앞에서부터 하나씩 뒤지기)과 비교하면 하늘과 땅 차이입니다!

5️⃣ 실전 응용: 파이썬 `bisect` 라이브러리 활용법 🛠️

직접 `while`문을 짜는 것도 좋지만, 파이썬에는 이미 초고속 이진 탐색 라이브러리인 bisect가 내장되어 있습니다.

📏 마스터 코치의 비유: 새치기 할 자리 찾기
bisect는 기본적으로 "이 숫자를 기존 정렬을 망치지 않고 끼워 넣는다면 몇 번째 인덱스에 들어가야 할까?"를 찾아주는 마법의 줄자입니다. 
import bisect

data = [10, 20, 30, 30, 30, 40, 50]

# 1. 값이 들어갈 위치 (왼쪽 기준)
# 30을 넣으려고 할 때, 기존 30들의 '맨 앞' 위치(인덱스 2)를 알려줌
left_idx = bisect.bisect_left(data, 30)
print(left_idx) # 2

# 2. 값이 들어갈 위치 (오른쪽 기준)
# 30을 넣으려고 할 때, 기존 30들의 '맨 뒤' 위치(인덱스 5)를 알려줌
right_idx = bisect.bisect_right(data, 30)
print(right_idx) # 5

# 꿀팁! 중복된 30의 개수 구하기!! (오른쪽 인덱스 - 왼쪽 인덱스)
counts = right_idx - left_idx
print(f"30은 총 {counts}개 있습니다.") # 3개

📌 핵심 요약

  • 이진 탐색 (Binary Search): 정렬된 데이터에서 정중앙을 찌르며 절반씩 버리는 고속 탐색 기법.
  • 전제 조건: 데이터는 무조건 정렬(Sort)되어 있어야 한다.
  • 시간 복잡도: $O(\log N)$. 100만 개 데이터도 20번이면 끝!
  • bisect 라이브러리: 파이썬에서 제공하는 이진 탐색 모듈. 인덱스 찾을 때 bisect_left, bisect_right를 적극 활용하자.
