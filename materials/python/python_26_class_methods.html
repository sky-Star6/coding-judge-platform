💜 파이썬 마스터 코치: 26. 클래스 메서드

클래스의 숨겨진 능력치 해제! 다양한 종류의 스킬(메서드)을 배워봅시다. ⚔️

1️⃣ 인스턴스 메서드 (Instance Methods) 기본 스킬

우리가 앞서 배운 가장 기본적인 형태의 메서드입니다. 객체(인스턴스)가 있어야만 사용할 수 있습니다.

🗡️ 마스터 코치의 비유: 내 개인 스킬
검사(객체)가 검을 휘두르는 것(메서드)처럼, 항상 나 자신(self)을 기준으로 행동합니다. 
class Warrior:
    def __init__(self, name):
        self.name = name

    # 인스턴스 메서드 (첫 번째 인자는 무조건 self)
    def attack(self):
        print(f"{self.name}이(가) 공격합니다!")

hero = Warrior("아서스")
hero.attack() # 아서스이(가) 공격합니다!

2️⃣ 클래스 메서드 (Class Methods) 길드 스킬

개인(객체)이 아니라, 클래스 전체(설계도 자체)가 사용하는 메서드입니다. @classmethod 장식을 붙여줍니다.

📢 마스터 코치의 비유: 길드 공지사항
개인 능력치가 아니라, 우리 "전사 길드" 전체와 관련된 일을 처리할 때 씁니다.
self 대신, 클래스 자신을 의미하는 cls를 첫 번째 인자로 받습니다. 
class Warrior:
    guild_name = "전사 길드" # 이건 클래스 변수 (모두가 공유)

    @classmethod
    def print_guild(cls):
        # cls는 클래스 자체(Warrior)를 가리킵니다.
        print(f"우리는 {cls.guild_name} 소속이다!")

# 객체를 만들지 않고도 클래스 이름으로 바로 부를 수 있습니다!
Warrior.print_guild() # 우리는 전사 길드 소속이다!

3️⃣ 정적 메서드 (Static Methods) 독립 유틸리티

클래스 안에 있기는 하지만, 클래스(cls)나 객체(self)의 데이터를 전혀 사용하지 않는 독립적인 함수입니다. @staticmethod 장식을 붙입니다.

🧮 마스터 코치의 비유: 길드 앞 계산기
길드 안에 놓여있긴 한데, 그냥 아무나 와서 숫자 계산만 하고 가는 단순한 기계입니다. self도, cls도 필요 없습니다. 
class MathTool:
    @staticmethod
    def add(a, b):
        return a + b # self나 cls가 아예 없습니다!

# 객체 생성 없이 바로 사용 가능
result = MathTool.add(10, 20)
print(result) # 30

4️⃣ 특수 메서드 (Magic/dunder Methods) 숨겨진 마법

이름 양쪽에 언더스코어 두 개(__)가 붙은 스페셜 메서드들입니다. 우리가 직접 부르지 않아도, 특정 상황에서 자동으로 실행됩니다.

🪄 마스터 코치의 비유: 마법 주문
• __init__: "태어날지어다!" (생성될 때)
• __str__: "정체를 밝혀라!" (print()로 출력할 때) 
class Book:
    def __init__(self, title):
        self.title = title

    # print() 함수에 넣었을 때 어떻게 보여줄지 결정하는 마법!
    def __str__(self):
        return f"<책: {self.title}>"

my_book = Book("파이썬 기초")
print(my_book) # <책: 파이썬 기초>  (만약 __str__이 없다면 메모리 주소가 나옵니다)

5️⃣ 비공개 메서드 (Private Methods) 비밀 번호

이름 앞에 __ (언더스코어 두 개)를 붙이면, 클래스 밖에서는 절대 부를 수 없는 비밀 메서드(또는 변수)가 됩니다.

class SecretSafe:
    def __init__(self):
        self.__password = "1234" # 접근 불가 변수

    def __unlock(self): # 접근 불가 메서드
        print("금고가 열렸습니다.")

    # 오직 클래스 내부의 다른 메서드를 통해서만 접근 가능
    def open_safe(self, pwd):
        if pwd == self.__password:
            self.__unlock()
        else:
            print("비밀번호 틀림!")

safe = SecretSafe()
# safe.__unlock() # 🚨 에러! 외부에서 직접 호출 불가능
safe.open_safe("1234") # 금고가 열렸습니다.

6️⃣ 추상 메서드 (Abstract Methods) 빈 강제 계약서

뼈대만 만들어 놓고, 이걸 상속받는 자식 클래스들에게 "너네가 무조건 내용 채워서 구현해!" 라고 강제하는 메서드입니다.

(사용하려면 abc 모듈을 import 해야 합니다)

from abc import ABC, abstractmethod

class Animal(ABC): # 추상 클래스 표시
    @abstractmethod
    def speak(self):
        pass # 내용은 비워둡니다 (자식들이 채워야 함)

class Dog(Animal):
    # 자식인 Dog는 반드시 speak()를 만들어야만 에러가 안 납니다!
    def speak(self):
        print("멍멍!")

dog1 = Dog()
dog1.speak()

  • a. @staticmethod
  • b. @classmethod
  • c. @abstractmethod
  • d. @instance
2. [코딩하기] 강제 계약서 서명하기

아래에서 Bird 클래스가 추상 클래스 Animal을 상속받았습니다. 어떤 메서드를 구현하지 않아서 에러가 날까요? 빈칸을 채워 에러를 없애보세요.

from abc import ABC, abstractmethod
class Animal(ABC):
    @abstractmethod
    def cry(self):
        pass

class Bird(Animal):
    def _________(self):
        print("짹짹!")
🔑 정답 확인하기 (클릭)
  1. b: @classmethod는 첫 번째 인자로 cls를 받아 클래스 전체 문맥을 다룹니다.
  2. 코딩 2: 부모가 강제한 이름 그대로 cry를 빈칸에 채워 넣어야 합니다.

📌 핵심 요약

  • 인스턴스 메서드(기본): 객체의 데이터를 바꿀 때 (self 필요).
  • 클래스 메서드(@classmethod): 클래스 전체의 데이터를 바꿀 때 (cls 필요).
  • 정적 메서드(@staticmethod): 아무 데이터도 쓰지 않는 단순 도구.
  • 특수 메서드(__매직__): 파이썬이 알아서 부르는 마법 (ex. __str__).
  • 비공개 메서드(__이름): 밖에서 못 부르게 숨김 보호 기능.
  • 추상 메서드(@abstractmethod): 자식에게 구현을 강제하는 빈 설계도.
