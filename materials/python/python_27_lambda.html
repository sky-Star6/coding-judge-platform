✳️ 파이썬 마스터 코치: 27. 람다(Lambda) 함수

이름도 필요 없는 일회용 초고속 함수! 한 줄의 마법을 📝 써봅시다.

1️⃣ 람다 함수 기본 구성: 일회용 포스트잇 📝

def를 써서 정식으로 함수를 만들면 번거로울 때, 이름 없이(익명) 한 줄로 짧게 만드는 함수입니다.

📝 마스터 코치의 비유: 포스트잇 vs 정식 문서
def 함수가 도장이 찍힌 '정식 결재 서류'라면,
lambda 함수는 쓰고 바로 버리는 '포스트잇 메모지' 입니다!
lambda 매개변수 : 리턴할 결과식 
# 정식 함수 (def)
def plus_ten(x):
    return x + 10

# 람다 함수 (lambda) - 위와 완전히 똑같은 기능!
# x를 받아서, x+10을 돌려줘라
lambda_plus_ten = lambda x: x + 10

print(lambda_plus_ten(5)) # 15

2️⃣ `map`과 단짝 친구 🤝

람다가 진짜 힘을 발휘하는 건 다른 함수(특히 내장 함수)와 섞어 쓸 때입니다. 리스트의 모든 요소를 일괄 변경할 때 map()을 씁니다.

nums = [1, 2, 3]

# 리스트의 모든 숫자에 100을 더하고 싶다면?
# map(적용할 함수, 리스트)
result = list(map(lambda x: x + 100, nums))

print(result) # [101, 102, 103]

3️⃣ `filter`로 거름망 돌리기 🧪

리스트에서 특정 조건(True/False)을 만족하는 것만 뽑아낼 때 filter()를 씁니다.

nums = [1, 2, 3, 4, 5]

# 짝수(x를 2로 나눈 나머지가 0인 것)만 골라내기
evens = list(filter(lambda x: x % 2 == 0, nums))

print(evens) # [2, 4]

4️⃣ `sorted`의 기준(key)으로 사용하기 🗂️

파이썬 고수들이 가장 많이 쓰는 패턴입니다! 복잡한 데이터를 내 맘대로 정렬할 때 씁니다.

words = ["apple", "pie", "banana"]

# 그냥 정렬하면 알파벳 순 (a -> b -> p)
print(sorted(words)) # ['apple', 'banana', 'pie']

# '단어의 길이(len)'를 기준으로 정렬하고 싶다면?
# 정렬 기준(key) 자리에 포스트잇(람다)을 붙여줍니다!
short_to_long = sorted(words, key=lambda x: len(x))

print(short_to_long) # ['pie', 'apple', 'banana'] 길이 순!

5️⃣ 조건부 표현식(if)과 함께 사용하기 🚦

람다 안에서도 if ~ else를 쓸 수 있습니다. 단, 람다의 문법은 참일때값 if 조건 else 거짓일때값 순서입니다.

# 점수가 80 이상이면 "Pass", 아니면 "Fail"
check_score = lambda score: "Pass" if score >= 80 else "Fail"

print(check_score(90)) # Pass
print(check_score(50)) # Fail

6️⃣ 매개변수 2개 이상 사용하기 👥

콤마(,)를 사용해서 인자를 여러 개 받을 수도 있습니다.

# 두 수 a, b를 받아서 곱하는 람다 함수
multiply = lambda a, b: a * b

print(multiply(3, 4)) # 12

📌 핵심 요약

  • 람다(Lambda): 한 줄짜리 이름 없는 간편한 함수. 포스트잇! 📝
  • 문법: lambda 매개변수: 리턴값
  • 주요 활용처: map(), filter(), sorted(key=...) 안에서 조건으로 던져줄 때.
  • def를 쓸 곳에는 웬만하면 람다를 쓸 수 있지만, 너무 복잡하면 def를 쓰는 것이 보기 좋습니다.
