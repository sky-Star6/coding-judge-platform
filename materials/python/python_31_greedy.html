💰 파이썬 마스터 코치: 31. 그리디 알고리즘

[논리적 사고 III] 가장 똑똑한 선택, '그리디(Greedy) 알고리즘'

1️⃣ 탐욕적 선택: 지금 당장 가장 좋아 보이는 것을 고르기 🍩

그리디(Greedy) 알고리즘(탐욕법)이란 현재 상황에서 "지금 당장 눈앞에 확실하게 보이는 최선의 선택"을 하는 방법입니다.

🍕 마스터 코치의 비유: 피자 조각 고르기
눈앞에 피자 여러 판이 있습니다. 가장 많이 먹으려면 어떻게 해야 할까요? 전체 피자를 다 스캔할 필요 없이, 가장 큰 조각부터 얼른 집어드는 것입니다.
미래를 계산하지 않고 "현재 가장 맛있는(이득이 큰) 선택"을 반복하는 것이 그리디의 핵심입니다.

2️⃣ 정당성 확인: "항상 최선의 결과가 나올까?" 질문 던지기 🤔

"당장의 눈앞의 선택이 항상 완벽한 결과를 가져올까?" 아쉽게도 꼭 그렇지는 않습니다. 하지만 어떤 문제들은 그리디 방식만으로도 항상 100점짜리 완벽한 정답을 보장합니다.

이것을 '그리디가 정당한가?'(정당성 확인)라고 부릅니다. 그리디를 사용하기 전에, 이것이 문제를 푸는 진짜 정답이 맞는지 비판적으로 고민해야 합니다.

3️⃣ 대표 문제 실습: 거스름돈 문제와 회의실 배정 전략 💵

그리디가 100% 정답을 보장하는 대표적인 두 가지 유형을 살펴보겠습니다.

# 거스름돈 거슬러주기 (동전 개수 최소화)

손님에게 1,260원을 거슬러 주어야 합니다. 500원, 100원, 50원, 10원짜리 동전이 있을 때 "가장 큰 금액의 동전부터 최대한 많이" 주는 것이 당연히 동전 개수를 줄이는 길입니다.

n = 1260
count = 0
coin_types = [500, 100, 50, 10] # 무조건 큰 단위부터!

for coin in coin_types:
    count += n // coin  # 해당 동전으로 거슬러 줄 수 있는 개수 세기
    n %= coin           # 남은 돈 계산하기

print(f"최소 동전 개수: {count}개") # 2 + 2 + 1 + 1 = 6개

💡 정당성: 큰 동전(500원)이 항상 작은 동전(100원)들의 배수 형태이므로, 무조건 큰 거부터 쓰는 게 이득이기 때문에 그리디가 성립합니다.

4️⃣ 정렬과의 시너지: 그리디를 위해 먼저 정렬하기 🔄

그리디 문제를 풀 때 가장 좋은 짝꿍은 바로 정렬(Sort)입니다. "가장 큰 것", "가장 먼저 끝나는 것"을 선택해야 하므로, 미리 데이터를 정렬해 두면 선택하기가 훨씬 수월합니다.

# 회의실 배정 문제

회의실은 1개인데, 여러 팀이 사용하고 싶어 합니다. 가장 많은 팀이 회의실을 쓰게 하려면? "빨리 끝나는 회의"부터 차례대로 배정하면 됩니다!

# [시작 시간, 끝나는 시간]
meetings = [[1, 4], [3, 5], [0, 6], [5, 7], [3, 8], [5, 9], [6, 10], [8, 11], [8, 12], [2, 13], [12, 14]]

# 핵심 포인트! 회의가 끝나는 시간 기준으로 먼저 정렬합니다.
meetings.sort(key=lambda x: (x[1], x[0]))

count = 0
end_time = 0

for meet in meetings:
    if meet[0] >= end_time: # 내 시작 시간이 앞선 회의의 종료 시간보다 같거나 늦다면
        end_time = meet[1]  # 회의를 진행하고, 종료 시간 업데이트!
        count += 1

print(f"진행 가능한 최대 회의 수: {count}개")

5️⃣ 한계점: 그리디가 실패하는 순간과 DP의 필요성 🚫

우리를 무적처럼 만들어 줄 것 같던 그리디 알고리즘도 실패하는 순간(반례)이 존재합니다.

⚠️ 큰 단위가 작은 단위의 배수가 아닐 때!
만약 동전이 500원, 400원, 100원이 있고, 거스름돈 800원을 줘야 한다고 칩시다.
- 그리디의 선택 (큰 거부터): 500원 1개 + 100원 3개 = 총 4개
- 가장 완벽한 선택 (최적해): 400원 2개 = 총 2개

이처럼 눈앞의 500원에 눈이 멀면 전체 결과가 손해를 보는 구조에서는 그리디가 먹히지 않습니다!

이런 경우에는 모든 가능성을 체계적으로 기록하며 풀어가는 동적 계획법(DP, Dynamic Programming)을 배워야 합니다.

change = 170
# 그리디가 작동하려면 어떤 순서로 리스트를 적어야 할까요?
coin_types = [________, ________, ________]

count = 0
for coin in coin_types:
    count += change // coin
    change %= coin

print(f"동전 개수: {count}개")  # 정답은 4개가 나와야 합니다. (100*1 + 50*1 + 10*2)
