❇️ 파이썬 마스터 코치: 20. 함수 만들기

복잡한 코드를 깔끔하게 정리하는 마법 상자(자판기)를 만들어봅시다!

1️⃣ 함수(Function) 만들기: 자판기 설치 🎰

자주 쓰는 코드를 묶어서 이름을 붙여둔 것입니다. 필요할 때마다 이름만 부르면 실행됩니다.

🎰 마스터 코치의 비유: 자판기
매번 커피를 직접 타 먹는 건 귀찮죠?
"커피 자판기"를 한 번 만들어두면, 버튼만 누르면 됩니다.
(def = 기계를 정의한다 / 만든다) 
# 함수 정의 (자판기 설치)
def hello():
    print("안녕하세요!")
    print("반갑습니다.")

# 함수 호출 (버튼 누르기)
hello()
hello() # 여러 번 재사용 가능!

2️⃣ 매개변수(Parameter): 동전 넣기 🪙

함수가 작동하는 데 필요한 재료(데이터)를 외부에서 넣어줄 수 있습니다.

🪙 마스터 코치의 비유: 동전 투입구
자판기에 "500원"을 넣느냐 "1000원"을 넣느냐에 따라
나오는 음료수가 달라지거나 잔돈이 달라지겠죠? 
# name이라는 구멍(매개변수)이 있는 함수
def hello_friend(name):
    print(name, "님, 안녕하세요!")

# 데이터를 던져주며 호출!
hello_friend("철수")
hello_friend("영희")

# 출력:
# 철수 님, 안녕하세요!
# 영희 님, 안녕하세요!

3️⃣ 반환값(return): 음료수 나오기 🥤

함수가 일을 다 마치고, 그 결과물을 나에게 다시 돌려주는 것입니다.

🥤 주의! print vs return

print: 화면에 글씨만 보여주고 끝남. (내가 가질 수 없음)

return: 진짜 물건(데이터)을 뱉어냄. (변수에 담아서 쓸 수 있음!)

def add(a, b):
    result = a + b
    return result  # result를 밖으로 던져줌!

# 반환된 값을 sum_val 변수에 받아서 저장
sum_val = add(3, 5)
print("결과:", sum_val) # 결과: 8

4️⃣ 값 여러 개 반환하기: 종합 세트 🍱

파이썬에서는 콤마(,)를 이용해 여러 개의 결과를 한 번에 돌려줄 수 있습니다. (사실 튜플로 묶여서 나옵니다)

def calculate(a, b):
    plus = a + b
    minus = a - b
    return plus, minus  # 두 개를 동시에!

# 1. 튜플로 한 번에 받기
result = calculate(10, 5)
print(result)  # (15, 5)

# 2. 따로따로 받기 (Unpacking)
p, m = calculate(10, 5)
print("더하기:", p) # 15
print("빼기:", m)   # 5

5️⃣ 함수 호출 과정: 주문과 배달 🧾

코드는 위에서 아래로 실행되지만, 함수를 호출하면 점프했다가 다시 돌아옵니다.

  1. def를 만나면: "아, 이런 기계가 있구나" 하고 기억만 하고 지나감.
  2. 함수 호출(call())을 만나면: 그 함수 코드로 점프!
  3. 함수 코드 실행.
  4. return을 만나거나 끝까지 가면: 호출했던 곳으로 복귀!

6️⃣ 변수의 유효 범위(Scope): 내 구역 vs 공공 구역 🏠🌏

변수가 어디서 만들어졌느냐에 따라 사용할 수 있는 범위가 달라집니다.

🏠 마스터 코치의 비유: 내 방 vs 거실

지역 변수(Local): 내 방(함수) 안에 있는 일기장. 거실(밖)에서는 볼 수 없음!

전역 변수(Global): 거실(코드 전체)에 있는 시계. 어디서든 볼 수 있음!

global_var = "전역변수(거실 시계)"

def my_room():
    local_var = "지역변수(내 일기장)"
    print(global_var) # 거실 시계는 방에서도 보임 (O)
    print(local_var)  # 내 일기장은 내가 봄 (O)

my_room()
print(global_var) # 밖에서 거실 시계 봄 (O)
# print(local_var) # 에러! 남의 방 일기장은 밖에서 안 보임 (X)

7️⃣ 함수 실행 원리: 스택 프레임(Stack Frame) 🥞

함수가 호출될 때마다 메모리에 작업 공간(프레임)이 차곡차곡 쌓입니다.

🥞 마스터 코치의 비유: 접시 쌓기 (Stack)
함수를 부를 때마다 새 접시(프레임)를 위에 올리고 작업을 합니다.
작업이 끝나면(return) 맨 위 접시부터 치웁니다.
🎬 상황극: 함수가 함수를 부를 때
  • 1. main(전역) 접시가 바닥에 깔림.
  • 2. hello() 호출 → 그 위에 hello 접시가 쌓임.
  • 3. hello 작업 끝 → 접시 치워짐(사라짐).
  • 4. 다시 main 접시만 남음.
