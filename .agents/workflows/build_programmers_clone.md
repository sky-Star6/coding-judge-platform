# 🚀 프로그래머스 클론 플랫폼 개발 워크플로우 (최종 통합본)

이 워크플로우는 프로그래머스와 유사한 종합 온라인 코딩 테스트 플랫폼을 파이썬(Flask)과 바닐라 프론트엔드로 구축하기 위한 실제 적용 명세서입니다. 본 프로젝트는 총 29단계를 거쳐 백엔드 채점 엔진, 웹 API, 그리고 모바일 반응형 프론트엔드까지 완성형 구조를 띄고 있습니다.

## 🗄️ 1. 데이터베이스 및 백엔드 (Backend API & DB)
사용자 데이터와 문항 정보, 제출 이력을 보관하고 통신하는 주축입니다.
- **DB 스키마 (`judge_db.sqlite`)**: `users`(인적사항, 권한등급), `problems`(문제 유형, 난이도, 초기 템플릿 코드), `test_cases`(숨김 처리 가능), `submissions`(채점 내역 및 상태). 스키마 확장(v1~v8)을 거쳐 설계됨.
- **Flask API (`app.py`)**: 세션(Session) 기반 로그인 관리. 문제 검색 필터링(`GET`), 코드 제출 및 채점 비동기 포워딩(`POST`), 유저 권한(레벨 1~3, admin) 계층별 라우팅 제어.

## ⚖️ 2. 핵심 채점 엔진 (`simple_judge.py`)
제출물을 독립적으로 실행시키고 자원(메모리/시간)을 통제하며 엣지 케이스를 방어합니다.
- **파이프라인 통제 메커니즘**: `subprocess` 를 활용하여 Python 코드와 Java 소스를 각각 컴파일/실행하고 입출력 스트림(`stdin/stdout`) 캡처.
- **상태 판별기**: 제한 시간(Time Limit) 초과 시 `TLE`, 런타임/컴파일 에러 시 `RE/CE`, 정답/오답 통과율 산정(`AC/WA`).
- **보안 방어선 (Anti-Cheat)**: 파이썬 코드 문자열 검사를 통해 `os`, `sys`, `subprocess`, `exec`, `eval` 등의 백도어 해킹 키워드가 발견되면 즉시 `CE(보안위반)` 처분.

## 💻 3. 프론트엔드 인터페이스 (Frontend UI)
학생들이 훈련하고 제출 결과를 피드백 받는 반응형 인터페이스입니다.
- **홈 / 명예의 전당 (`index.html`)**: 난이도 배지 시스템, 레벨 및 진도율 필터, 실시간 랭킹 리더보드 노출, 모바일(스마트폰) 세로 화면 최적화.
- **코딩 작업장 (`judge.html`)**:
  - `Split.js` 창 분할 라이브러리 탑재 (좌측: 문제 설명, 우측 상단: 코드 에디터, 우측 하단: 실행 콘솔창). 모바일 단말기 진입 시 자동으로 세로(Vertical) 스태킹 뷰로 전환.
  - 마크다운 파서 및 `MathJax` 수식 기호 동적 렌더링 포함.
  - 마이크로소프트 **Monaco Editor** 탑재 (구문 강조, 폰트 조절, 초기 코드 템플릿 복구 버튼).
- **어드민 패널 (`admin_*.html`)**: 문제 신규 출제, 학생 계정 비밀번호/권한 덮어쓰기 기능 및 학생별 전체 제출 로그 조회 모달 창. 모바일 좌우 스와이프 그리드 방어 적용.

## 🧩 4. 특수 문제 유형 및 커스텀 제약 시스템
단순한 '전체 코드 작성' 외에도 다양한 훈련 모드 보안 시스템을 갖추고 있습니다.
- **빈칸 채우기 (Fill-in-the-blank)**: 
  - 에디터 내부에 `___` (언더바 3개 이상) 영역을 식별하여 시각적으로 노란색 형광펜 매핑.
  - `onKeyDown` 이벤트 매니저가 커서 좌표를 추적하여 빈칸 밖의 문자열 조작이나 엔터(줄바꿈) 등 코드 파괴 행위를 원천 방어 (가상 Read-Only Lock).
- **디버깅 훈련 (Debugging)**:
  - 파이썬 내장 `difflib.ndiff` 엔진이 서버단에서 작동. 초기 코드 템플릿과 제출문을 '줄 단위(Line diff)' 로 뜯어 비교.
  - 2줄을 초과해 수정한 흔적(차이점 4점 이상)이 포착되면 결과가 맞더라도 즉각 채점을 차단하고 **"RV (조건위반: 3줄 이상 수정됨)"** 강제 반환.
