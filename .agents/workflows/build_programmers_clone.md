# 프로그래머스 플랫폼 개발 워크플로우

이 워크플로우는 프로그래머스와 유사한 온라인 코딩 테스트 및 알고리즘 연습 플랫폼을 단계별로 구축하기 위한 체계적인 지침입니다. 현재 작성 중이신 `simple_judge.py` (채점 엔진) 및 `judge_db.sqlite` (데이터베이스)를 기반으로 확장하는 구조입니다.

## 1단계: 데이터베이스 설계 및 초기화 (Database Setup)
채점 시스템에 필요한 핵심 데이터를 저장할 구조를 잡습니다.
- [ ] **필수 테이블 설계 및 생성:**
  - `users`: 사용자 정보 (ID, 비밀번호, 닉네임 등)
  - `problems`: 문제 정보 (문제 번호, 제목, 마크다운 설명, 난이도, 시간 제한, 메모리 제한)
  - `test_cases`: 문제별 테스트 데이터 (문제 번호, 입력값, 예상 출력값, 공개/비공개 여부)
  - `submissions`: 제출 기록 (사용자 ID, 문제 번호, 제출한 코드 언어, 코드 내용, 채점 상태, 소요 시간, 메모리 사용량)
- [ ] 기본적인 더미 데이터(테스트용 문제와 테스트 케이스) 삽입 스크립트 작성.

## 2단계: 핵심 채점 엔진 고도화 (Judge Engine Core)
보안과 정확성이 가장 중요한 단계입니다. 제출된 코드를 안전하게 실행하고 채점합니다.
- [ ] **코드 실행 래퍼(Wrapper) 구현 (`simple_judge.py` 확장):**
  - 제출된 코드를 임시 파일로 생성.
  - Python의 `subprocess` 모듈 등을 사용하여 외부 프로세스로 실행.
  - 시스템 입력(`stdin`)으로 테스트 케이스를 넘겨주고, 시스템 출력(`stdout`)을 캡처.
- [ ] **자원 제한 및 상태 판별:**
  - 시간 제한 설정 및 `TimeoutError` 예외 처리 (시간 초과 판정).
  - 결과값을 비교하여 상태코드 반환 (정답: `AC`, 오답: `WA`, 런타임에러: `RE`, 시간초과: `TLE`).
- [ ] **(권장) 샌드박스 환경 격리:** Docker 컨테이너를 사용하여 서버 시스템(Host) 파일에 접근하지 못하도록 격리 코드 추가.

## 3단계: 백엔드 API 서버 개발 (Backend API)
클라이언트(웹 브라우저)와 채점 엔진을 연결해주는 다리 역할을 합니다. (FastAPI, Flask, Node.js 등 활용)
- [ ] 데이터베이스와 통신하여 플랫폼 데이터를 조회/추가하는 로직 연동.
- [ ] REST API 엔드포인트 구현:
  - `GET /api/problems` : 문제 목록 불러오기.
  - `GET /api/problems/{id}` : 특정 문제의 상세 정보와 기본 제공(템플릿) 코드 불러오기.
  - `POST /api/submissions` : 사용자 코드 제출 및 비동기 채점 요청.
- [ ] (필요시) 채점 대기열을 처리할 큐(Message Queue, 예: Celery) 시스템 도입.

## 4단계: 직관적인 프론트엔드 UI 구축 (Frontend Web UI)
프로그래머스 고유의 편리한 '스플릿 뷰(Split View)' 인터페이스를 웹 화면으로 만듭니다.
- [ ] **화면 레이아웃 구성:**
  - **좌측 패널:** 문제 설명(Markdown 렌더링 지원).
  - **우측 상단 패널:** 코드 에디터 (Monaco Editor 혹은 CodeMirror 라이브러리 연동으로 구문 강조 및 자동 완성 제공).
  - **우측 하단 패널:** '실행 결과' 및 '테스트 케이스' 관리 콘솔.
- [ ] 백엔드 API와 비동기 통신(Fetch/Axios) 연동하여 '코드 실행' 버튼과 '제출 후 채점하기' 버튼 동작 구현.
- [ ] 채점 진행 과정(로딩 애니메이션, 통과 퍼센티지 등)을 시각적으로 피드백.

## 5단계: 통합 테스트 및 폴리싱 (Integration & Polishing)
- [ ] 전체 파이프라인(UI 제출 -> API 서버 데이터 변환 -> 채점 엔진 샌드박스 실행 -> API 서버 결과 수신 -> UI 결과 표시) 테스트.
- [ ] 다양한 엣지 케이스 확인: 무한 루프 코드를 제출했을 때 서버가 마비되지 않는지, 메모리 초과 코드를 잘 걸러내는지 검증.
